## 7 函数接口（续）

### 7.11 性能采样（续）

#### 7.11.4 性能采样结果（续）

打印输出多个 SPE 的性能采样数据。不同 SPE 的数据理论上相同，实际执行时根据代码的执行情况可能会存在微小差异，具体请以实际情况为准。

```text
Kernel perf data of thread 0:
 --------------------------------------------------------------------------------------
|START LINE NUM                     |7                                                 |
|LATENCY                            |11013                                             |
|CNT_L1IC_MISS                      |0                                                 |


Kernel perf data of thread 1:
 --------------------------------------------------------------------------------------
|START LINE NUM                     |7                                                 |
|LATENCY                            |11013                                             |
|CNT_L1IC_MISS                      |0                                                 |
....


Kernel perf data of thread 31:
 --------------------------------------------------------------------------------------
|START LINE NUM                     |7                                                 |
|LATENCY                            |11013                                             |
|CNT_L1IC_MISS                      |0                                                 |
```

**输出说明**

| 字段 | 说明 |
|------|------|
| START LINE NUM | 测试开始的行号。 |
| LATENCY | 延迟时间，即执行周期，单位：拍。<br>说明：具体结果可能随着不同的硬件上下波动。 |
| CNT_L1IC_MISS | 指令缓存脱靶次数，单位：次。 |

#### 7.11.5 clock

```c
unsigned long clock()
```

**功能介绍**

获取当前 SPE 内时钟周期计数器的值。您可以在代码段前后进行两次插桩采样，并计算两次采样的差值，统计指定代码段的执行周期。

**参数解释**

无。

**返回值**

| 返回值类型 | 说明 |
|------------|------|
| unsigned long | 返回当前 SPE 内时钟周期计数器的值。 |

**注意事项**

- 需要手动包含 `sdaa_clock.h` 头文件。
- 需要引入 sdaa 命名空间。

**使用示例**

以统计某段循环加法代码的执行周期为例，介绍如何使用 `clock`。

```c
__device__ void sum_func(const int *arr1, const int *arr2, int len, int *res_arr)
{
    // 在指定代码段之前插桩，获取起始时钟周期计数器的值
    unsigned long start_time = clock();


    for (int i = 0; i < len; i++) {
        res_arr[i] = arr1[i] + arr2[i];
    }


    // 在指定代码段之后插桩，获取起始时钟周期计数器的值
    unsigned long end_time = clock();


    // 打印代码段的执行周期
    printf("SPE ID: %lu, timd: %lu\n", threadIdx, end_time - start_time);
}
__global__ void test()
{
    int *res_arr = (int *)malloc(SIZE * sizeof(int));
    int array1[SIZE];
    int array2[SIZE];
    for (int i = 0; i < SIZE; i++) {
        array1[i] = i;
        array2[i] = i * 10;
    }
    sum_func(array1, array2, SIZE, res_arr);
    free(res_arr);
}


int main()
{
    sdaaSetDevice(0);
    test<<<1>>>();
    sdaaDeviceSynchronize();
    return 0;
}
```

### 7.13 程序调试

#### 7.13.1 概述

程序调试是应用程序开发人员在进行软件开发过程中必要的纠错手段。一款优秀的程序调试工具可以协助软件开发人员快速定位代码在编译、运行时发生错误的原因和位置，提升软件的开发效率。

SDAA C 提供了如下程序调试接口：

- `printf`：打印设备端运行时日志。通过打印运行时日志的方式，追踪打印计算核心 SPE 中代码的运行状态，从而定位出问题发生的具体原因。
- `abort`：终止设备端程序。通过终止设备端程序的方式，在设备端的函数代码中检测到无法修复的问题时，及时停止设备端程序的运行。
- `assert`：断言。通过断言的方式，检测代码中存在的错误，出现错误时打印对应的文件行号等详细信息。

#### 7.13.2 printf

```c
int printf(const char *format, ...)
```

**功能介绍**

打印设备端运行时日志。可以通过调用 `printf` 接口打印设备端目标变量的运行时状态。

**参数解释**

| 参数名称 | 说明 |
|----------|------|
| format | 格式化字符串，指定需要输出的文本及格式化说明符。<br>说明：常用格式化说明符：<br>`%d`：打印有符号十进制整数。<br>`%u`：打印无符号十进制整数。<br>`%f`：打印浮点数（默认 6 位小数）。<br>`%c`：打印单个字符。<br>`%s`：打印字符串。<br>`%x`/`%X`：打印十六进制整数（小写/大写）。<br>`%p`：打印指针地址。<br>`%%`：打印百分号。 |
| … | 可变参数，对应格式化说明符的变量或值。 |

**返回值**

- 打印成功，返回打印的字符数。
- 打印失败，返回负值。

**注意事项**

- `printf` 接口是 SDAA C 提供的内置接口，使用时无需引入头文件。
- 设置环境变量 `export SDAA_SYNC_PRINT=1`，可以按照 SPE 编号从小到大的顺序进行打印。默认按照 SPE 的执行顺序进行打印。

**使用示例**

```c
__device__ void sum_func(int a, int b)
{
    int g_sum = a + b;
    printf("threadIdx: %lu, sum: %d\n", threadIdx, g_sum);
}


__global__ void bar()
{
    sum_func(1, 2);
}


int main()
{
    sdaaSetDevice(0);
    bar<<<1>>>();
    sdaaDeviceSynchronize();


    return 0;
}
```

- 不设置环境变量 `SDAA_SYNC_PRINT` 或者值默认为 0 时，会按照 SPE 的执行顺序进行打印。执行后打印结果如下：

```text
threadIdx: 4, sum: 3
threadIdx: 0, sum: 3
threadIdx: 6, sum: 3
threadIdx: 7, sum: 3
......
```

- 设置环境变量 `SDAA_SYNC_PRINT=1`，按照 SPE 编号从小到大的顺序进行打印。执行后打印结果如下：

```text
threadIdx: 0, sum: 3
threadIdx: 1, sum: 3
threadIdx: 2, sum: 3
threadIdx: 3, sum: 3
......
```

#### 7.13.3 abort

```c
void abort()
```

**功能介绍**

终止设备端程序。如果您在设备端编写的函数代码中，检查到无法修复的问题，可以通过 `abort` 接口直接终止设备端的程序运行。

**参数解释**

无。

**返回值**

无。

**注意事项**

- `abort` 接口是 SDAA C 提供的内置接口，使用时无需引入头文件。
- 通过 `abort` 函数结束运行时：
  - 如果设置了环境变量 `SDAA_ENABLE_COREDUMP_ON_EXCEPTION`（详情请参考《SDAARuntime 用户手册》的 `SDAA_ENABLE_COREDUMP_ON_EXCEPTION` 章节），通知主机端生成包含错误相关上下文的 Core Dump 文件，并且终止主机端的代码执行。
  - 如果没有设置该环境变量 `SDAA_ENABLE_COREDUMP_ON_EXCEPTION`（详情请参考《SDAARuntime 用户手册》的 `SDAA_ENABLE_COREDUMP_ON_EXCEPTION` 章节）：继续执行主机端的代码，后续通过执行 `sdaaDeviceSynchronize` 接口，获取对应的 abort 错误码。

**使用示例**

以检测除数不为 0 作为示例，当检查到除数为 0 时，执行 `abort` 终止 SPE 后续程序的运行。

```c
__device__ int g_sum = 0;
__device__ int my_div(int a, int b)
{
    if (b == 0) {
        // 如果除数b为0，终止SPE后续程序的运行
        abort();
    }
    return a / b;
}


__device__ void my_compute(int a, int b)
{
    int div_result = my_div(a, b);
    g_sum = (a - b) * div_result;
}


__global__ void bar()
{
    my_compute(1, 2);
}


int main()
{
    sdaaSetDevice(0);
    bar<<<1>>>();
    sdaaDeviceSynchronize();


    return 0;
}
```

#### 7.13.4 assert

```c
void assert(bool condition)
```

**功能介绍**

检查代码中是否出现错误，出现错误后会打印对应错误所在的文件行号等信息，之后终止设备端程序的运行。

**参数解释**

| 参数名称 | 说明 |
|----------|------|
| condition | 需要检查的条件表达式。<br>说明：如果条件表达式的返回值为 0/false，则触发断言失败。 |

**返回值**

无。

**注意事项**

- `assert` 接口是 SDAA C 提供的内置接口，使用时无需引入头文件。
- 通过 `assert` 函数终止设备端程序运行时：
  - 如果设置了环境变量 `SDAA_ENABLE_COREDUMP_ON_EXCEPTION`（详情请参考《SDAARuntime 用户手册》的 `SDAA_ENABLE_COREDUMP_ON_EXCEPTION` 章节）：通知主机端生成包含错误相关上下文的 Core Dump 文件，并且终止主机端的代码执行。
  - 如果没有设置环境变量 `SDAA_ENABLE_COREDUMP_ON_EXCEPTION`（详情请参考《SDAARuntime 用户手册》的 `SDAA_ENABLE_COREDUMP_ON_EXCEPTION` 章节）：继续执行主机端的代码，后续通过执行 `sdaaDeviceSynchronize` 接口，获取对应的 assert 错误码。

**使用示例**

以检测除数不为 0 作为示例，`assert` 会检查除数是否为 0，为 0 时会打印错误所在的文件行号信息并且终止 SPE 后续程序的执行。

```c
__device__ int g_sum = 0;
__device__ void my_div(int a, int b)
{
    // 如果除数b为0，则打印错误信息并终止SPE后续程序的执行
    assert(b != 0 && "b should not be zero");
    g_sum = a / b;
}


__global__ void bar()
{
    my_div(1, 2);
}


int main()
{
    sdaaSetDevice(0);
    bar<<<1>>>();
    sdaaDeviceSynchronize();


    return 0;
}
```

## 8 数学函数

### 8.1 概述

数学函数不需要 `#include` 相应的头文件即可使用。SDAA C 内置了包括：指数计算、对数计算、正弦计算、余弦计算、正切计算、高斯误差计算等常用标量数学计算接口。为了满足用户对于应用程序运行性能的追求，SDAA C 还提供了基于 ULP（Unit in the Last Place）精度标识的向量数学计算接口。

### 8.2 单精度标量接口

#### 8.2.1 ceilf

```c
float ceilf(float x)
```

**功能介绍**

计算不小于给定单精度浮点数 x 的最小整数。最大 ULP 为 0。

**参数解释**

| 参数名称 | 说明 |
|----------|------|
| x | 输入参数。 |

**返回值**

| 返回值类型 | 说明 |
|------------|------|
| float | 大于等于参数 x 的最小整数。<br>假设 y 为小于 x 的最大整数，当 x 位于 (y, y + 1] 的时候，返回 y + 1。<br>当 x 取 ±∞ 或者 NaN 时，返回 x。 |

**注意事项**

该接口支持原位操作。

**使用示例 1**

```c
__device__ void test(float *a, float *b, int len)
{
    for (int i = 0; i < len; i++) {
        b[i]= ceilf(a[i]) + 1;
    }
}
```

**使用示例 2**

```c
__device__ void test(float *a, int len)
{
    for (int i = 0; i < len; i++) {
        a[i]= ceilf(a[i]) + 1;    // 原位操作
    }
}
```
