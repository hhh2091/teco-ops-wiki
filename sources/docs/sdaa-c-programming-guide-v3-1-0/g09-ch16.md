## 16.3（续）下三角方程组求解优化

承接上文，负责对连续 16 个待求解方程进行消元的计算过程也被封装为一个独立的 device 函数：

```cpp
__device__ void eliminate_16x16(float* lt_blk, float* x, float* b) {
    floatv16 vec_b[2];
    vec_b[0] = vec_b[1] = 0.0f;

    for (int j = 0; j < 16; j += 2) {
        // expand to vector
        floatv16 vec_x[2];
        vec_x[0] = -x[j];
        vec_x[1] = -x[j + 1];

        // load vector
        floatv16 vec_lt[2];
        simd_load(vec_lt[0], lt_blk + j * 16);
        simd_load(vec_lt[1], lt_blk + (j + 1) * 16);

        // FMA
        vec_b[0] += vec_lt[0] * vec_x[0];
        vec_b[1] += vec_lt[1] * vec_x[1];
    }

    floatv16 vres;
    simd_load(vres, b);
    vec_b[0] += vec_b[1];
    vres += vec_b[0];
    simd_store(vres, b);

    // over
    return;
}
```

这里为了充分利用 SPE 的浮点算力，使用了向量指令加速消元过程。同时，分配了两组向量寄存器，构成两组数据无依赖的计算流水线，手动进行循环展开。

### 优化效果

在 SPE 主频为 2.36GHz 的硬件上测试，对一个 2048 阶的下三角方程组进行求解时，最终优化版本相较于初步优化版本的加速比约为 15.6。

## 16.4 算子调优实战

### 16.4.1 GEMM 基础实现

#### 16.4.1.1 实现原理

本文以 GEMM（General Matrix Multiply，通用矩阵乘法）为例，介绍如何使用不同的优化方法，如：SIMD，矩阵乘法单元，广播，双缓冲等方法提升 GEMM 算子性能。

本章主要介绍未使用优化方法的 GEMM 算子的基础实现（仅用单线程进行计算），效率低且耗时长，性能不佳。

##### 计算原理

GEMM 用于执行两个矩阵的乘法，并将结果累加到第三个矩阵上。

##### 计算公式

```
C = alpha × op(A) × op(B) + beta × C
```

其中：

- `alpha` 和 `beta` 是标量，`A`、`B`、`C` 是矩阵，`op()` 是可选的转置操作，本示例中：矩阵 A 和矩阵 B 均未转置，alpha 与 beta 取值分别为 1 和 0。
- 矩阵 A 的维度是 M×K、矩阵 B 的维度是 K×N、矩阵 C 的维度是 M×N。

图1 GEMM 矩阵乘法

##### 设备端计算流程

1. 读取输入数据：将数据从 Global 存储空间读取到 SPM 存储空间。
2. 矩阵计算：在 SPM 存储空间，进行矩阵乘计算。
3. 传输计算结果：将矩阵乘的计算结果，从 SPM 存储空间传输到 Global 存储空间。

##### 性能优化目标

设备端计算流程的总用时最少，即：读取输入数据、矩阵乘计算和传输计算结果的总用时最少。

##### 基础实现

基础实现是通过三层嵌套循环，依次在 M、N 和 K 维度遍历矩阵 A 和矩阵 B 的元素，按照矩阵乘法的规则计算结果，从而得到矩阵 C。

以下是 GEMM 基础实现的代码实现：

1. 参数 `GEMMArgs` 中存放矩阵 `A`、`B`、`C` 的维度信息和存储步长信息。
   a. 维度信息：描述矩阵的大小，即矩阵的行数和矩阵的列数；
   b. 存储步长信息：描述使用二维数组在内存中存储矩阵时，同一列中相邻元素之间的偏移量；
2. 将矩阵 `A`、`B`、`C` 的指针类型转换为 `half` 类型。
3. 使用三层嵌套的 `for` 循环，执行矩阵乘法操作：
   - 最外层的 `for` 循环，遍历矩阵 `A` 的行，索引为 `idx`。
   - 中间层的 `for` 循环，遍历矩阵 `B` 的列，索引为 `idy`。
   - 最内层的 `for` 循环，遍历矩阵 `A` 的列和矩阵 `B` 的行，索引为 `idz`。并将矩阵 `A` 的第 `idx` 行和矩阵 `B` 的第 `idy` 列对应元素相乘，将结果累加到 `temp` 中。
4. `temp` 的结果，存储到矩阵 `C` 的 `idx` 行、`idy` 列位置。

示例代码如下：

```cpp
__device__ void tecoKernelGemmFT16SingleThreadImpl(GEMMArgs pGemm)
{
    // 矩阵的维度
    int M, N, K;
    // 矩阵的存储步长
    int lda, ldb, ldc;
    // 指向矩阵 A、B、C 的指针
    const half *pA, *pB;
    half *pC;
    // 存储中间计算结果
    float temp;

    // 从 GEMMArgs 结构体中获取矩阵的维度信息
    M = pGemm.m;
    N = pGemm.n;
    K = pGemm.k;
    // 获取矩阵 A、B、C 的存储步长
    lda = pGemm.lda;
    ldb = pGemm.ldb;
    ldc = pGemm.ldc;
    // 将矩阵 A、B、C 的指针转换为 half 类型
    pA = (const half *)pGemm.A;     // [M][K]
    pB = (const half *)pGemm.B;     // [K][N]
    pC = (half *)pGemm.C;           // [M][N]
    int idx, idy, idz;

    // 遍历矩阵 A 的行
    for (idx = 0; idx < M; idx++) {
        // 遍历矩阵 B 的列
        for (idy = 0; idy < N; idy++) {
            // 初始化临时结果为 0
            temp = 0;
            // 进行矩阵乘法的核心计算，将矩阵 A 的第 idx 行与矩阵 B 的第 idy 列元素相乘并累加
            for (idz = 0; idz < K; idz++) {
                temp += (float)pA[idx * lda + idz] * (float)pB[idz * ldb + idy];
            }
            // 将计算结果存储到矩阵 C 的对应位置
            pC[idx * ldc + idy] = (half )temp;
        }
    }
}
```

#### 16.4.1.2 完整程序

GEMM 基础实现的完整程序（`gemm_base.scpp`）：

```cpp
#include <sdaa_perf.h>
#include <type_traits>

using namespace sdaa;

// GEMMArgs包含矩阵乘法所需的参数
typedef struct GEMMArgs {
    int m;    // 矩阵 A 与矩阵 C 的行数
    int n;    // 矩阵 B 与矩阵 C 的列数
    int k;    // 矩阵 A 的列数与矩阵 B 的行数
    int lda;    // 矩阵 A 的列间距
    int ldb;    // 矩阵 B 的列间距
    int ldc;    // 矩阵 C 的列间距
    int bM;    // 分块矩阵的行数
    int bN;    // 分块矩阵的列数
    int bK;    // 分块矩阵的中间维度
    const void *A;    // 矩阵 A 的指针
    const void *B;    // 矩阵 B 的指针
    void *C;    // 矩阵 C 的指针
} GEMMArgs;

__global__ void tecoKernelGemmImpl(GEMMArgs pGemm)
{
    // 矩阵的维度
    int M, N, K;
    // 矩阵的存储步长
    int lda, ldb, ldc;
    // 指向矩阵 A、B、C 的指针
    const half *pA, *pB;
    half *pC;
    // 存储中间计算结果
    float temp;

    // 从 GEMMArgs 结构体中获取矩阵的维度信息
    M = pGemm.m;
    N = pGemm.n;
    K = pGemm.k;
    // 获取矩阵 A、B、C 的存储步长
    lda = pGemm.lda;
    ldb = pGemm.ldb;
    ldc = pGemm.ldc;

    // 开始性能采样
    perf_start();

    pA = (const half *)pGemm.A;   // [M][K]
    pB = (const half *)pGemm.B;   // [K][N]
    pC = (half *)pGemm.C;         // [M][N]
    int idx, idy, idz;

    // 遍历矩阵 A 的行
    for (idx = 0; idx < M; idx++) {
        // 遍历矩阵 B 的列
        for (idy = 0; idy < N; idy++) {
            // 初始化临时结果为 0
            temp = 0;
            // 进行矩阵乘法的核心计算，将矩阵 A 的第 idx 行与矩阵 B 的第 idy 列元素相乘并累加
            for (idz = 0; idz < K; idz++) {
                temp += (float)pA[idx * lda + idz] * (float)pB[idz * ldb + idy];
            }
            // 将计算结果存储到矩阵 C 的对应位置
            pC[idx * ldc + idy] = (half)temp;
        }
    }

    // 性能采样
    perf_stop();
    // 打印出性能采样结果
    perf_print();
}

int main()
{
    sdaaSetDevice(0);

    GEMMArgs args;

    // 设置矩阵维度参数
    args.m = 1024;
    args.n = 256;
    args.k = 1024;

    args.lda = args.k;
    args.ldb = args.n;
    args.ldc = args.n;
    args.bM = 512;
    args.bN = 256;
    args.bK = 256;

    // 在主机上为矩阵 A、B 和 C 分配内存
    half *h_A = (half *)malloc(args.m * args.k * sizeof(half));
    half *h_B = (half *)malloc(args.k * args.n * sizeof(half));
    half *h_C = (half *)malloc(args.m * args.n * sizeof(half));

    // 为矩阵 A、B 和 C 设置初始值
    for (size_t i = 0; i < args.m * args.k; ++i) {
        h_A[i] = static_cast<half>(1);
    }

    for (size_t i = 0; i < args.k * args.n; ++i) {
        h_B[i] = static_cast<half>(1);
    }

    for (size_t i = 0; i < args.m * args.n; ++i) {
        h_C[i] = static_cast<half>(0);
    }

    // 在设备上为矩阵 A、B 和 C 分配内存
    half *d_A, *d_B, *d_C;

    sdaaMalloc((void **)(&d_A), args.m * args.k * sizeof(half));
    sdaaMalloc((void **)(&d_B), args.k * args.n * sizeof(half));
    sdaaMalloc((void **)(&d_C), args.m * args.n * sizeof(half));

    sdaaMemcpy(d_A, h_A, args.m * args.k * sizeof(half), sdaaMemcpyHostToDevice);
    sdaaMemcpy(d_B, h_B, args.k * args.n * sizeof(half), sdaaMemcpyHostToDevice);
    sdaaMemcpy(d_C, h_C, args.m * args.n * sizeof(half), sdaaMemcpyHostToDevice);

    args.A = d_A;
    args.B = d_B;
    args.C = d_C;

    tecoKernelGemmImpl<<<1>>>(args);

    sdaaDeviceSynchronize();

    // 将结果矩阵 C 从设备内存复制到主机内存
    sdaaMemcpy(h_C, d_C, args.m * args.n * sizeof(half), sdaaMemcpyDeviceToHost);

    // 释放设备端内存
    sdaaFree(d_A);
    sdaaFree(d_B);
    sdaaFree(d_C);

    // 释放主机端内存
    free(h_A);
    free(h_B);
    free(h_C);

    return 0;
}
```

### 16.4.2 GEMM 优化策略

#### 16.4.2.1 SIMD

##### 16.4.2.1.1 实现原理

本章主要介绍如何在 GEMM 算子的基础实现上，使用 SIMD 优化策略，提升算子性能。

###### 优化原理

SIMD（Single Instruction Multiple Data）单指令多数据，一条指令可以执行多个数据。

SPE 上有多个向量寄存器，支持向量指令加速。可以将计算过程中的标量操作转换为向量操作，通过 SIMD 向量处理的能力实现多数据并行操作。

图1：单指令多数据（SIMD）

在 GEMM 实现的过程中，数据加载和计算过程可以通过 SIMD 优化策略，将计算过程中常见的标量操作转换为向量操作，从而对数据进行批量处理，提升 GEMM 的性能。

###### 代码逻辑

SIMD 优化通过 `tecoKernelGemmSIMDImpl` 函数实现，该函数使用 SIMD 指令集高效地执行矩阵乘法操作。

1. 从参数 `GEMMArgs` 中获取矩阵 `A`、`B`、`C` 的维度信息和存储步长信息。
   a. 维度信息：描述矩阵的大小，即矩阵的行数和矩阵的列数；
   b. 存储步长信息：描述使用二维数组在内存中存储矩阵时，同一列中相邻元素之间的偏移量；
2. 使用 `for` 循环，遍历矩阵的分块：
   a. 根据 `idx` 计算当前分块的行索引 `idM` 和列索引 `idN`，并获取 `pCurrC` 的位置。
   b. 对每个 `K` 维度的分块：
      i. 获取矩阵 `A` 和 `B` 中当前分块的位置 `pCurrA` 和 `pCurrB`。
      ii. 使用 `memcpy_stride` 将数据从 Global 存储空间复制到 SPM 存储空间，并对 `LocalB` 进行转置存储在 `LocalB_T` 中。
      iii. 嵌套的 `for` 循环遍历 SPM 存储空间内矩阵分块后的行和列，使用 `reduce_Sdot` 计算矩阵乘法。
         - `reduce_Sdot` 第一个 `for` 循环处理长度为 64 的元素块，每次迭代加载并计算 64 个元素的点积，分四组 16 个元素进行处理，每组使用 `simd_load` 加载数据，计算并累加点积结果。
         - 第二个 `for` 循环处理长度为 16 的元素块，使用 `simd_load` 加载数据，计算并累加点积结果。
         - 最后合并中间结果。
      iv. 结果存储在 `TempC` 中并累加。
3. 根据结果矩阵的类型，将结果存储在 `LocalC` 中。
4. 将结果复制到矩阵 `C` 的对应位置，并释放分配的 SPM 存储空间。

示例代码如下：

```cpp
// 点积计算的宏定义
// 该宏根据数据长度分不同阶段进行点积计算
// 先处理长度大于等于 64 的部分，然后处理长度大于等于 16 的部分，最后将部分结果累加
#define unroll_dot(load_func_a, load_func_b, type, type_v) \
    do {                                                   \
        type *px1 = (type *)x1;                            \
        type *px2 = (type *)x2;                            \
        i = 0;                                             \
        int up;                                            \
        type_v a0, a1, a2, a3;                             \
        type_v b0, b1, b2, b3;                             \
        up = len - 64;                                     \
        for (; i <= up; i += 64) {                         \
            load_func_a(a0, px1 + 0);                      \
            load_func_a(a1, px1 + 16);                     \
            load_func_a(a2, px1 + 32);                     \
            load_func_a(a3, px1 + 48);                     \
            load_func_b(b0, px2 + 0);                      \
            load_func_b(b1, px2 + 16);                     \
            load_func_b(b2, px2 + 32);                     \
            load_func_b(b3, px2 + 48);                     \
            ans_v0 += a0 * b0;                             \
            ans_v1 += a1 * b1;                             \
            ans_v2 += a2 * b2;                             \
            ans_v3 += a3 * b3;                             \
            px1 += 64;                                     \
            px2 += 64;                                     \
        }                                                  \
        up = len - 16;                                     \
        for (; i <= up; i += 16) {                         \
            load_func_a(a0, px1 + 0);                      \
            load_func_b(b0, px2 + 0);                      \
            ans_v0 += a0 * b0;                             \
            px1 += 16;                                     \
            px2 += 16;                                     \
        }                                                  \
        ans_v0 += ans_v1;                                  \
        ans_v2 += ans_v3;                                  \
        ans_v0 += ans_v2;                                  \
    } while (0)

// 定义点积计算函数的宏定义
// 该宏根据指针的对齐情况选择不同的加载函数进行点积计算
// 最后将部分结果累加并处理剩余元素，得到最终点积结果
#define function_dot(func_name, type, type_v)                                               \
    static __device__ void func_name(const type *x1, const type *x2, int len, float *ans)   \
    {                                                                                       \
        int i = 0;                                                                          \
        type_v ans_v0 = 0, ans_v1 = 0, ans_v2 = 0, ans_v3 = 0;                              \
        int align1 = is_aligned(x1, sizeof(type_v));                                        \
        int align2 = is_aligned(x2, sizeof(type_v));                                        \
        if (align1) {                                                                       \
            if (align2)                                                                     \
                unroll_dot(simd_load, simd_load, type, type_v);                             \
            else                                                                            \
                unroll_dot(simd_load, simd_loadu, type, type_v);                            \
        } else {                                                                            \
            if (align2)                                                                     \
                unroll_dot(simd_loadu, simd_load, type, type_v);                            \
            else                                                                            \
                unroll_dot(simd_loadu, simd_loadu, type, type_v);                           \
        }                                                                                   \
                                                                                            \
        float tans = 0;                                                                     \
        floatv16 f_ans_v0;                                                                  \
        f_ans_v0 = ans_v0;                                                                  \
        for (int k = 0; k < 16; ++k) {                                                      \
            tans += ((float *)&f_ans_v0)[k];                                                \
        }                                                                                   \
        for (; i < len; ++i) {                                                              \
            tans += x1[i] * x2[i];                                                          \
        }                                                                                   \
        *ans = tans;                                                                        \
    }

function_dot(reduce_Sdot, float, floatv16);

__global__ void tecoKernelGemmSIMDImpl(GEMMArgs pGemm)
{
    // 线程块的行索引和列索引
    const int rid = threadIdx / 8;
    const int cid = threadIdx % 8;

    // 从 GEMMArgs 结构体中获取矩阵的维度信息
    const int M = pGemm.m;
    const int N = pGemm.n;
    const int K = pGemm.k;

    // 从 GEMMArgs 结构体中获取矩阵 A、B、C 的存储步长
    const int lda = pGemm.lda;
    const int ldb = pGemm.ldb;
    const int ldc = pGemm.ldc;

    // 获取矩阵 A、B、C 的指针
    const half *A = (const half *)pGemm.A;
    const half *B = (const half *)pGemm.B;
    half *C = (half *)pGemm.C;

    // 外部分块的大小
    const int bM = pGemm.bM;
    const int bN = pGemm.bN;
    const int bK = pGemm.bK;

    // 外部分块的数量
    int nM = M / bM;
    int nN = N / bN;
    int nK = K / bK;

    // 内部块的大小
    int LocalbM = bM / 8;
    int LocalbN = bN / 4;
    int LocalbK = bK;

    // 内部块的位置
    const half *pA = A + cid * LocalbM * lda;
    const half *pB = B + rid * LocalbN;
    half *pC = C + cid * LocalbM * ldc + rid * LocalbN;

    half *d_A, *d_B, *d_C;

    sdaaMalloc((void **)(&d_A), args.m * args.k * sizeof(half));
    sdaaMalloc((void **)(&d_B), args.k * args.n * sizeof(half));
    sdaaMalloc((void **)(&d_C), args.m * args.n * sizeof(half));

    sdaaMemcpy(d_A, h_A, args.m * args.k * sizeof(half), sdaaMemcpyHostToDevice);
    sdaaMemcpy(d_B, h_B, args.k * args.n * sizeof(half), sdaaMemcpyHostToDevice);
    sdaaMemcpy(d_C, h_C, args.m * args.n * sizeof(half), sdaaMemcpyHostToDevice);

    args.A = d_A;
    args.B = d_B;
    args.C = d_C;

    // 执行kernel函数
    tecoKernelGemmSIMDImpl<<<1>>>(args);

    sdaaDeviceSynchronize();

    // 将结果矩阵 C 从设备内存复制到主机内存
    sdaaMemcpy(h_C, d_C, args.m * args.n * sizeof(half), sdaaMemcpyDeviceToHost);

    // 释放设备端内存
    sdaaFree(d_A);
    sdaaFree(d_B);
    sdaaFree(d_C);

    // 释放主机端内存
    free(h_A);
    free(h_B);
    free(h_C);

    return 0;
}
```

#### 16.4.2.2 矩阵乘法单元

##### 16.4.2.2.1 实现原理

本章主要介绍如何在 SIMD 优化策略的基础上，使用矩阵乘法单元，提升算子性能。

###### 优化原理

矩阵乘法单元是专门用于执行矩阵乘加运算的模块。通过矩阵乘法单元实现的矩阵乘加计算，比依靠处理器的通用乘加指令实现的乘加计算更加高效。

在 GEMM 算法里，存在大量的矩阵乘加计算，因此若能合理地使用矩阵乘法单元，可以显著提升浮点运算的性能。

SDAA C 从调用接口上可分为阻塞型矩阵乘和非阻塞型矩阵乘。非阻塞型矩阵乘虽然使用相对复杂，但性能较好。因此，本文将使用非阻塞型矩阵乘进行性能优化，非阻塞型矩阵乘接口的详细使用方法，可参考非阻塞型矩阵乘的使用逻辑章节。

图1 阻塞型矩阵乘接口的使用逻辑

###### 代码逻辑

非阻塞型矩阵乘通过 `tecoKernelGemmMatmulImpl` 函数实现，该函数在 SIMD 的基础上使用矩阵乘法单元进行性能优化。

1. 根据 `threadIdx` 指定矩阵的行索引 `rid` 和列索引 `cid`。
2. 从参数 `GEMMArgs` 中获取矩阵 `A`、`B`、`C` 的维度信息和存储步长信息。
   a. 维度信息：描述矩阵的大小，即矩阵的行数和矩阵的列数；
   b. 存储步长信息：描述使用二维数组在内存中存储矩阵时，同一列中相邻元素之间的偏移量；
3. 分配 SPM 存储空间，包括 `LocalCompA`、`LocalCompB`、`LocalTB`、`LocalC`、`LocalCompC` 和 `tempC`。
4. 配置 `MatmulHandle` 接口参数，并初始化矩阵乘计算单元。
5. 使用 `for` 循环遍历矩阵的分块：
   a. 根据 `idx` 计算当前分块的行索引 `idM` 和列索引 `idN`，并获取 `pCurrA` 和 `pCurrB` 的位置。
   b. 等待矩阵乘法句柄加载矩阵 `A` 和 `B`，将数据复制到 SPM 存储空间并进行 SIMD 操作。
   c. 使用 `matmul_compute` 进行矩阵乘法计算操作。
   d. 使用 `matmul_wait` 等待运算结果写回。
6. 对结果 `C` 进行置换操作，使用 SIMD 操作存储结果。
7. 根据结果矩阵的类型决定是否进行数据类型转换，将结果存储在 `LocalC` 中。
8. 将结果复制到矩阵 `C` 的对应位置，并释放分配的 SPM 存储空间。
