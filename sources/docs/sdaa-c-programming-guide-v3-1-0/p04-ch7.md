##### 7.8.3.2.5 matmul_wait

```cpp
void matmul_wait(MatmulHandle &handle)
```

**功能介绍**

等待矩阵乘写回操作完成。

**参数解释**

| 参数名称 | 说明 |
|----------|------|
| handle | MatmulHandle 型变量。配置矩阵乘的参数，从而使用不同的矩阵乘功能。 |

**返回值**

无。

**注意事项**

- 需要手动包含 `sdaa_matmul.h` 头文件。
- 需要引入 sdaa 命名空间。

**使用示例1**

使用 matmul_init、matmul_load_weight、matmul_compute、matmul_store、matmul_wait、matmul_wait_loading_weight、matmul_wait_loading_input 共同完成 100\*32\*32 的矩阵乘运算。

```cpp
#include <sdaa_matmul.h>
using namespace sdaa;
__global__ void func()
{
    ... // 准备矩阵数据

    // 创建MatmulHandle类型变量，使用默认参数
    MatmulHandle handle;

    // 初始化矩阵乘接口并配置数据类型
    matmul_init(handle, MatmulHalfToFloat);

    // 加载权重矩阵，32*32个half型数据
    matmul_load_weight(handle, weight, MatmulK32, MatmulN32);

    // 等待权重矩阵加载完成
    matmul_wait_loading_weight(handle);

    ... // 此时可修改weight地址上数据

    // 加载输入矩阵，100*32个half型数据，并开始矩阵乘运算
    matmul_compute(handle, input, 100, MatmulK32);

    // 等待输入矩阵加载完成
    matmul_wait_loading_input(handle);

    ... // 此时可修改input地址上数据

    // 输出计算结果写回到指定地址，100*32个float型数据
    matmul_store(handle, output, 100, MatmulN32);

    // 等待写回完成
    matmul_wait(handle);
}
```

使用 matmul_init、matmul_load_weight、matmul_compute、matmul_store、matmul_wait、matmul_wait_loading_weight、matmul_wait_loading_input、matmul_set_updating_weight 共同完成 200\*32\*32 的矩阵乘运算。

**使用示例3**

使用 matmul_init、matmul_load_weight、matmul_compute、matmul_store、matmul_wait、matmul_wait_loading_weight、matmul_wait_loading_input、matmul_set_flushing_output 共同完成 100\*64\*32 的矩阵乘运算。

    // 等待权重矩阵加载完成
    matmul_wait_loading_weight(handle);

    ... // 此时可修改weight地址上数据

    // 等待输入矩阵加载完成
    matmul_wait_loading_input(handle);

    ... // 此时可修改input地址上数据

    // 输出计算结果写回到指定地址，100*32个float型数据

#### 7.8.3.3 配置型操作

##### 7.8.3.3.1 MatmulHandle

```cpp
MatmulHandle::MatmulHandle()
MatmulHandle::MatmulHandle(bool is_updating_weight, bool is_flushing_output, unsigned long weight_row_padding_mask = 0, MatmulOutputRowOffsetFlag output_row_offset_flag = MatmulEnableOutputRowOffset, unsigned int output_row_offset = 0)
```

**功能介绍**

配置矩阵乘的参数，从而使用不同的矩阵乘功能。创建 MatmulHandle 类型的变量时：

- 如果不提供参数，则使用其默认值。
- 如果提供参数，则使用参数指定的值配置矩阵乘。每个参数均设有对应的配置型函数接口。

**参数解释**

| 参数名称 | 说明 |
|----------|------|
| is_updating_weight | 是否更新权重矩阵（默认更新），对应函数接口 matmul_set_updating_weight：<br>- true：更新权重矩阵，切换权重矩阵加载缓冲区，使用新加载的权重矩阵进行下一次矩阵乘运算。<br>- false：不更新权重矩阵，不切换权重矩阵加载缓冲区，使用之前计算时使用的权重矩阵进行下一次矩阵乘运算。 |
| is_flushing_output | 是否在下一次计算后输出计算结果（默认输出），对应函数接口 matmul_set_flushing_output：<br>- true：下一次矩阵乘输出计算结果。<br>- false：下一次矩阵乘不输出计算结果。 |
| weight_row_padding_mask | 加载权重矩阵时填充空行的掩码，默认值为 0，对应函数接口 matmul_set_weight_row_padding。 |
| output_row_offset_flag | 枚举型 MatmulOutputRowOffsetFlag，设置计算结果行偏移标志位（默认为 MatmulEnableOutputRowOffset），对应函数接口 matmul_set_output_row_offset：<br>- MatmulEnableOutputRowOffset：使用指定的参数 output_row_offset 设置计算结果的行偏移。<br>- MatmulDisableOutputRowOffset：紧接着上一次计算结果之后的位置写入累加缓冲区。 |
| output_row_offset | 仅当 output_row_offset_flag 设置为 MatmulEnableOutputRowOffset 时有效，对应函数接口 matmul_set_output_row_offset。有效值：0~127，对应累加缓冲区条目数，默认值为 0。 |

**注意事项**

- 需要手动包含 `sdaa_matmul.h` 头文件。
- 需要引入 sdaa 命名空间。

**使用示例1**

```cpp
#include <sdaa_matmul.h>
using namespace sdaa;
__global__ void func()
{
    MatmulHandle handle; // 创建MatmulHandle型变量，使用默认参数

    ... // 执行非阻塞型矩阵乘操作
}
```

**使用示例2**

```cpp
#include <sdaa_matmul.h>
using namespace sdaa;
__global__ void func()
{
    // 创建MatmulHandle类型变量，不使能下一次计算结果输出，不使能高精度计算结果输出
    MatmulHandle handle(false, false);

    ... // 执行非阻塞型矩阵乘操作
}
```

##### 7.8.3.3.2 matmul_set_weight_row_padding

```cpp
void matmul_set_weight_row_padding(MatmulHandle &handle, unsigned long weight_row_padding_mask)
```

**功能介绍**

通过掩码的方式，设置加载权重矩阵时，是否填充空行，默认不填充。

**参数解释**

| 参数名称 | 说明 |
|----------|------|
| handle | MatmulHandle 型变量。配置矩阵乘的参数，从而使用不同的矩阵乘功能。 |
| weight_row_padding_mask | 加载权重矩阵时填充空行的掩码，默认值为 0。掩码的每一个 bit 位对应权重矩阵的一行：<br>- 为 0 时：不填充空行，加载权重矩阵对应的行。<br>- 为 1 时：填充空行，不加载权重矩阵的对应行。 |

**返回值**

无。

**注意事项**

- 需要手动包含 `sdaa_matmul.h` 头文件。
- 需要引入 sdaa 命名空间。

**使用示例**

```cpp
#include <sdaa_matmul.h>
using namespace sdaa;
__global__ void func()
{
    // 创建MatmulHandle类型变量，使用默认参数
    MatmulHandle handle;
    // 设置权重的mask，把权重矩阵的第1行屏蔽掉
    matmul_set_weight_row_padding(handle, 0x01);

    ... // 执行非阻塞型矩阵乘操作，加载权重矩阵时第1行填充0，权重矩阵从第2行开始写入
}
```

##### 7.8.3.3.5 matmul_set_flushing_output

```cpp
void matmul_set_flushing_output(MatmulHandle &handle, bool is_flushing_output)
```

**功能介绍**

无。

- 需要手动包含 `sdaa_matmul.h` 头文件。
- 需要引入 sdaa 命名空间。

#include <sdaa_matmul.h>
using namespace sdaa;
__global__ void func()
{
    // 创建MatmulHandle类型变量，使用默认参数

## 7.9 转置操作

### 7.9.1 概述

转置操作是指对矩阵的行和列进行互换，即将原始矩阵的行变换为列，或将原始矩阵的列变换为行。

转置操作分为阻塞型转置操作和非阻塞型转置操作：

- 阻塞型转置操作：使用简单，但性能不佳。
- 非阻塞型转置操作：使用相对复杂，但性能较好。

**阻塞型转置操作**

- transpose：进行矩阵的转置操作，原始矩阵可以位于 Global 或 SPM 存储空间。不支持原位操作。

**非阻塞型转置操作**

- TransposeHandle：用于控制不同的非阻塞型转置操作组。
- transpose_async：非阻塞型转置操作接口。原始矩阵地址只能位于 Global 存储空间，目的地址只能位于 SPM 存储空间。不支持原位操作。
- transpose_wait：等待当前计算核心 SPE 上所有使用同一个 TransposeHandle 的非阻塞型转置操作完成。
- check_transpose_async：验证 transpose_async 输入参数的合法性。

### 7.9.2 阻塞型转置

#### 7.9.2.1 transpose

```cpp
void transpose(void *dst, const void *src, unsigned int height, unsigned int width, size_t element_size)
```

**功能介绍**

完成同一 SPA 内的 Global 和 Global 存储空间之间、同一 SPE 内的 SPM 和 SPM 存储之间以及同一 SPA 内的 Global 存储和 SPM 存储之间的阻塞型矩阵转置操作。

> 图1 阻塞型矩阵转置

**参数解释**

| 参数名称 | 说明 |
|----------|------|
| dst | 目的矩阵地址，可以为 Global 存储或 SPM 存储的地址。 |
| src | 源矩阵地址，可以为 Global 存储或 SPM 存储的地址。 |
| height | 源矩阵的行数。 |
| width | 源矩阵的列数。 |
| element_size | 矩阵的数据类型大小。目前仅支持数据类型大小为 2B（short 类型或者 half 类型）或者 4B（int 类型或者 float 类型）。 |

**返回值**

无。

**注意事项**

- 需要手动包含 `sdaa_transpose.h` 头文件。
- 需要引入 sdaa 命名空间。
- 源矩阵地址和目的矩阵地址需要按数据类型进行对齐。例如：矩阵数据类型是 int 类型，则要求按 4B 对齐，矩阵数据类型是 short 类型，则要求按 2B 对齐。
- 转置接口不支持原位操作。
- 当前 transpose 接口会额外占用 SPM 堆空间：
  - 当数据类型大小为 2B 时，额外占用 2KB 的 SPM 堆内存空间。
  - 当数据类型大小为 4B 时，额外占用 1KB 的 SPM 堆内存空间。

**使用示例**

```cpp
#include <sdaa_transpose.h>
using namespace sdaa;

#define SIZE 256      // 矩阵大小
#define HEIGHT 16     // 矩阵行数
#define WIDTH 16      // 矩阵列数

__device__ void func()
{
    // 仅使用0号SPE进行矩阵转置
    if (threadIdx != 0) {
         return;
    }

    int *src = (int *)malloc(SIZE * sizeof(int));
    int *dst = (int *)malloc(SIZE * sizeof(int));

    // 初始化源矩阵：
    // {0, 1, 2, ..., 15,
    //   16, 17, 18, ..., 31,
    //   ...,
    //   240, 241, 242, ..., 255}
    for (int row = 0; row < HEIGHT; row++) {
         for (int col = 0; col < WIDTH; col++) {
             src[row * WIDTH + col] = row * WIDTH + col;
         }
    }

    // 执行转置操作
    transpose(dst, src, HEIGHT, WIDTH, sizeof(int));

    // 打印转置结果
    // {0, 16, 32, ..., 240,
    //   1, 17, 33, ..., 241,
    //   ...,
    //   15, 31, 47, ..., 255}
    for (int row = 0; row < HEIGHT; row++) {
         for (int col = 0; col < WIDTH; col++) {
             printf("%d\t", dst[row * WIDTH + col]);
         }
         printf("\n");
    }
    printf("\n");

    free(dst);
    free(src);
}
```

#### 7.9.3.4 check_transpose_async

```cpp
unsigned int check_transpose_async(void *dst, const void *src, unsigned int height, unsigned int width, size_t element_size, MemoryStride stride, TransposeDirect direct, TransposeHandle &handle)
```

**功能介绍**

验证非阻塞 transpose_async 输入参数的合法性，通过返回值判断输入参数是否满足特定的输入要求，如：源地址与目的地址是否对齐、转置操作的数据量是否符合要求等。

**参数解释**

| 参数名称 | 说明 |
|----------|------|
| dst | 转置操作的目的矩阵地址。 |
| src | 转置操作的源矩阵地址。 |
| height | 目的矩阵的行数。<br>- 当数据类型大小为 2B 时，需为 32。<br>- 当数据类型大小为 4B 时，需为 16。 |
| width | 目的矩阵的列数。<br>- 当数据类型大小为 2B 时，需为 32。<br>- 当数据类型大小为 4B 时，需为 16。 |
| element_size | 矩阵的数据类型大小。 |
| stride | MemoryStride 结构体类型变量，设置源和目标存储空间的跨步单元大小，单位：Byte。<br>说明：<br>- MemoryStride 的设置受硬件限制：<br>&nbsp;&nbsp;- 目前仅支持在 Global 存储空间，设置跨步单元的大小。<br>&nbsp;&nbsp;- 跨步单元的大小，需要小于 Global 存储空间的大小，可参考存储空间可用大小。<br>- 如果 MemoryStride 中的 dst_stride 和 src_stride 都设置为 0，则为连续数据搬运。 |
| direct | 转置方向，枚举类型 TransposeDirect，目前支持：<br>- TransposeGlobalToSpm：源矩阵位于 Global 存储空间，目的矩阵位于 SPM 存储空间。 |
| handle | TransposeHandle 类型变量，用于控制不同的非阻塞型转置操作组。 |

**返回值**

| 返回值类型 | 说明 |
|------------|------|
| unsigned int | 参数合法性验证结果，每个 Bit 代表不同的状态信息。对应枚举型 MemcpyAsyncStatus，当前可表示的状态有：<br>- TRANSPOSE_ASYNC_SUCC：参数无误。<br>- TRANSPOSE_ASYNC_ERR_ADDR_UNALIGNED：源地址或目的地址不满足对齐要求。<br>- TRANSPOSE_ASYNC_ERR_SIZE_ILLEGAL：源矩阵的行数或者列数不满足要求。<br>- TRANSPOSE_ASYNC_ERR_DIRECT_ILLEGAL：转置方向不满足要求。<br>- TRANSPOSE_ASYNC_ERR_STRIDE_PARAM：跨步参数错误。 |

返回的具体状态信息，可参照如下表格：

| 第 0 位 | 第 1 位 | 第 2 位 | 第 3 位 | 第 4 位 |
|---------|---------|---------|---------|---------|
| 0：参数无误 | 第 0 位是 0 时，此位保留 | 第 0 位是 0 时，此位保留 | 第 0 位是 0 时，此位保留 | 第 0 位是 0 时，此位保留 |
| 1：参数有误 | 第 0 位是 1 时，此位：<br>0：正常。<br>1：错误。源地址或目的地址不满足对齐要求。 | 第 0 位是 1 时，此位：<br>0：正常。<br>1：错误。源矩阵的行数或者列数不满足要求。 | 第 0 位是 1 时，此位：<br>0：正常。<br>1：错误。转置方向不满足要求。 | 第 0 位是 1 时，此位：<br>0：正常。<br>1：错误。跨步参数错误。 |

**注意事项**

- 需要手动包含 `sdaa_transpose.h` 头文件。
- 需要引入 sdaa 命名空间。

**使用示例**

```cpp
#include <sdaa_transpose.h>
using namespace sdaa;

#define SIZE 256
#define HEIGHT 16
#define WIDTH 16

// 声明地址64B对齐的Global存储空间
__device__ int src[SIZE] __attribute__((aligned(64)));
__device__ void func()
{
    // 仅使用0号SPE进行矩阵转置
    if (threadIdx != 0) {
         return;
    }

    // 在SPM存储空间申请用于存放目的矩阵的内存空间
    int *dst = (int *)malloc(SIZE * sizeof(int));

    // 初始化源矩阵：
    // {0, 1, 2, ..., 15,
    //   16, 17, 18, ..., 31,
    //   ...,
    //   240, 241, 242, ..., 255}
    for (int row = 0; row < HEIGHT; row++) {
         for (int col = 0; col < WIDTH; col++) {
             src[row * WIDTH + col] = row * WIDTH + col;
         }
    }

    // 声明TransposeHandle类型变量
    TransposeHandle handle;

    // 如果transpose_async接口参数合法，check_transpose_async返回TRANSPOSE_ASYNC_SUCC
    unsigned int check_val = check_transpose_async(dst, src, HEIGHT, WIDTH, sizeof(int), {0, 0}, TransposeGlobalToSpm, handle);
    if (check_val == TRANSPOSE_ASYNC_SUCC) {
         // 执行转置操作
         transpose_async(dst, src, HEIGHT, WIDTH, sizeof(int), {0, 0}, TransposeGlobalToSpm, handle);

         // 其他代码逻辑

         // 等待完成非阻塞转置操作
         transpose_wait(handle);
    }

    // 打印转置结果
    // {0, 16, 32, ..., 240,
    //   1, 17, 33, ..., 241,
    //   ...,
    //   15, 31, 47, ..., 255}
    for (int row = 0; row < HEIGHT; row++) {
         for (int col = 0; col < WIDTH; col++) {
             printf("%d\t", dst[row * WIDTH + col]);
         }
         printf("\n");
    }
    printf("\n");

    // 释放申请的SPM存储空间
    free(dst);
}
```

## 7.10 向量操作

### 7.10.1 概述

SDAA C 提供的向量操作类函数接口通过封装 SIMD（Single Instruction Multiple Data）指令一次性处理多个数据，极大地提高了数据处理的并行性。用于向量操作的函数接口可以分为以下几类：

- 装入/存储类：描述标量和向量间的数据搬运接口。
- 运算类：描述整数、浮点向量类型的算术运算、比较和选择等运算接口。具体可细分为：
  - 标量扩展类
  - 比较选择类
  - 元素整理类
  - 符号拷贝类
  - 类型转换类
  - 算术运算类
  - 比较选择类
  - 融合乘加类

- 对于混合类型的计算，目前不支持使用操作符替代函数接口。例如，不支持以下操作：

  ```cpp
  floatv16 leftVal;
  float16v16 rightVal;
  floatv16 res = leftVal + rightVal;
  ```

- 对于非混合类型（同一类型）的计算，算术运算类和比较类接口可用操作符进行替代。目前支持的操作符详见下表。

  > 说明：
  >
  > 对于比较类函数接口，当使用操作符进行替代时，需要根据目标类型增加强制类型转换。例如：
  >
  > ```cpp
  > floatv16 leftVal;
  > floatv16 rightVal;
  > floatv16 res = floatv16(leftVal <= rightVal);
  > ```

| 类别 | 操作符 |
|------|--------|
| 下标运算符 | `[]` |
| 单目运算符 | `+, -, *, &, !, ~, sizeof` |
| 算术运算符 | `+, -, *, /, %` |
| 逻辑运算符 | `<, <=, >, >=, ==, !=, &, ^, |, &&, ||` |
| 赋值运算符 | `=, +=, -=, *=, /=, %=, &=, ^=, |=` |
