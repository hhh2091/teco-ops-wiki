## 13（续）性能调优 — 浮点融合示例（接上一部分）

> 本节是浮点融合（FMA 类）性能调优示例代码的延续。前文已定义 `test_fmas`/`test_fmss`/`test_fnmas`/`test_fnmss`（单精度）等 Kernel，本段从 `test_fnmsd`（双精度负乘减）Kernel 开始，给出余下的设备端 Kernel、主机端调用封装、测试函数与 `main`，最后给出三种 `-ffp-contract` 编译选项下的优化效果对比。

### 双精度负乘减 Kernel：`test_fnmsd`

该 Kernel 实现双精度的负乘减运算 `data[0] = -(data[1] * data[2]) - data[3]`，仅在 `threadIdx == 0` 的计算核心上收集性能数据。

```c
__global__ void test_fnmsd(double *data)
{
    if (threadIdx != 0) {
        return;
    }
    // 收集性能数据起始值
    perf_start();
    for (int i = 0; i < N; i++) {
        for (int j = 0; j < SIZE; j++) {
            data[0 * SIZE + j] = -((data[1 * SIZE + j]) * data[2 * SIZE + j]) -
                                  data[3 * SIZE + j];
        }
    }
    // 收集性能数据结束值
    perf_stop();
    // 输出性能数据
    perf_print();
}
```

> 说明：上一部分的 `test_fmad`/`test_fmsd`/`test_fnmad`（双精度乘加/乘减/负乘加）Kernel 与本 Kernel 结构一致，区别仅在融合运算的符号组合。

### 单精度调用封装：`call_kernel_float`

封装单精度四种融合运算（乘加 `fmas`、乘减 `fmss`、负乘加 `fnmas`、负乘减 `fnmss`）的设备内存分配、数据搬运、Kernel 调用与结果回传。

```c
void call_kernel_float(float *src, float (*output)[SIZE])
{
    float *dev_fmas = NULL;
    float *dev_fmss = NULL;
    float *dev_fnmas = NULL;
    float *dev_fnmss = NULL;
    // 设置使用0号SPA
    sdaaSetDevice(0);
    // 在Global内存中malloc一段空间
    sdaaMalloc((void **)(&dev_fmas), NUM * SIZE * sizeof(float));
    // 将数据从Host端拷贝到Device端
    sdaaMemcpy(dev_fmas, src, NUM * SIZE * sizeof(float),
               sdaaMemcpyHostToDevice);
    // 在Global内存中malloc一段空间
    sdaaMalloc((void **)(&dev_fmss), NUM * SIZE * sizeof(float));
    // 将数据从Host端拷贝到Device端
    sdaaMemcpy(dev_fmss, src, NUM * SIZE * sizeof(float),
               sdaaMemcpyHostToDevice);
    // 在Global内存中malloc一段空间
    sdaaMalloc((void **)(&dev_fnmas), NUM * SIZE * sizeof(float));
    // 将数据从Host端拷贝到Device端
    sdaaMemcpy(dev_fnmas, src, NUM * SIZE * sizeof(float),
               sdaaMemcpyHostToDevice);
    // 在Global内存中malloc一段空间
    sdaaMalloc((void **)(&dev_fnmss), NUM * SIZE * sizeof(float));
    // 将数据从Host端拷贝到Device端
    sdaaMemcpy(dev_fnmss, src, NUM * SIZE * sizeof(float),
               sdaaMemcpyHostToDevice);

    test_fmas<<<1>>>(dev_fmas);
    test_fmss<<<1>>>(dev_fmss);
    test_fnmas<<<1>>>(dev_fnmas);
    test_fnmss<<<1>>>(dev_fnmss);

    // 将数据从Device端拷贝到Host端
    sdaaMemcpy(output[0], dev_fmas, SIZE * sizeof(float),
               sdaaMemcpyDeviceToHost);
    sdaaMemcpy(output[1], dev_fmss, SIZE * sizeof(float),
               sdaaMemcpyDeviceToHost);
    sdaaMemcpy(output[2], dev_fnmas, SIZE * sizeof(float),
               sdaaMemcpyDeviceToHost);
    sdaaMemcpy(output[3], dev_fnmss, SIZE * sizeof(float),
               sdaaMemcpyDeviceToHost);

    // 释放申请的Global内存
    sdaaFree(dev_fmas);
    sdaaFree(dev_fmss);
    sdaaFree(dev_fnmas);
    sdaaFree(dev_fnmss);
}
```

### 单精度测试函数：`test_float`

在主机端用随机数初始化数据，主机端串行计算参考结果，调用 `call_kernel_float` 得到设备端结果，并以 `1e-2` 为阈值逐元素比较。

```c
void test_float()
{
    int errnum = 0;
    float data[NUM * SIZE];
    float dev_output[NUM][SIZE];
    float host_output[NUM][SIZE];

    // init data
    for (int i = 0; i < NUM * SIZE; i++) {
        data[i] = (((float)rand()) / (float)RAND_MAX) * (MAX - MIN) + MIN;
    }
    for (int i = 0; i < NUM; i++) {
        for (int j = 0; j < SIZE; j++) {
            dev_output[i][j] = 0;
            host_output[i][j] = 0;
        }
    }
    // calculate in host
    for (int i = 0; i < SIZE; i++) {
        host_output[0][i] = data[1 * SIZE + i] * data[2 * SIZE + i] +
                               data[3 * SIZE + i];
        host_output[1][i] = data[1 * SIZE + i] * data[2 * SIZE + i] -
                               data[3 * SIZE + i];
        host_output[2][i] = -(data[1 * SIZE + i] * data[2 * SIZE + i]) +
                               data[3 * SIZE + i];
        host_output[3][i] = -(data[1 * SIZE + i] * data[2 * SIZE + i]) -
                               data[3 * SIZE + i];
    }

    call_kernel_float(data, dev_output);

    for (int i= 0; i < NUM; i++) {
        for (int j = 0; j < SIZE; j++) {
            if (fabs(host_output[i][j] - dev_output[i][j]) > 1e-2) {
                   printf("host_output[%d][%d] = %f dev_output[%d][%d] = %f\n",
                          i, j, host_output[i][j], i, j, dev_output[i][j]);
                   errnum++;
            }
        }
    }

    if (!errnum) {
        printf("all pass in float!\n");
    } else {
        printf("error in float!\n");
    }
}
```

## 15. 入门实战

### 15.1 向量运算

#### 15.1.1 概述

本例使用 LTO 模式进行编译，在太初 AI 加速卡上实现向量相关的运算（包括向量装载、向量平方根运算、向量指数运算、向量乘、向量加、向量小于等于比较运算以及向量存储操作）。完整程序请参考完整程序。

主机端通过 SDAARuntime 接口分配设备端的 Global 存储空间，并将数据从主机端内存拷贝到设备端的 Global 存储空间。在设备端通过向量装载接口给向量赋值，通过向量平方根函数计算向量的平方根，通过指数运算函数对平方根运算的结果做指数运算，再通过 `*` 运算符和 `+` 运算符分别做向量乘和向量加运算，最后比较指数运算的结果是否小于等于向量乘和向量加运算的结果。在主机端通过 SDAARuntime 接口将 Global 存储空间上运算结果拷贝到主机端内存，并打印运算结果。

#### 15.1.2 编程思路

**主机端**

1. 分配主机端内存并赋值。
2. 主机端与设备端的数据传输。
   a. 设置计算核心阵列 SPA 号。

   c. 从主机端内存向 Global 存储空间传输数据。
   d. 调用 Kernel 函数。
   e. 从 Global 存储空间拷贝计算结果到主机端内存。
   f. 释放 Global 存储空间。
3. 打印运算结果。
4. 释放主机端内存。

**设备端**

1. 向量装载。
2. 向量平方根运算。
3. 向量指数运算。
4. 向量乘和向量加运算。
5. 向量小于等于比较。
6. 向量存储。

#### 15.1.3 编写程序

##### 15.1.3.1 主机端

**主机端程序设计思路**

1. 分配主机端内存并赋值：
   a. 在主机端通过 `malloc` 接口分配 2 个主机端存储空间 `h_data1`, `h_data2`，大小都是为 `SIZE`。其中：`SIZE` 值为 `16 * sizeof(float)`。
   b. 将已分配的 2 个存储空间的起始地址分别返回给指针 `*h_data1`, `*h_data2`。`*h_data1`, `*h_data2` 均为指向单精度数据类型的指针。
   c. 在 Host 端定义 3 个 const 常量 `COUNT`，`SIZE`，`TMPVAL`。其中：
      - `COUNT`：表示数据的个数。
      - `SIZE`：表示 `COUNT` 个单精度数据类型的字节大小。
      - `TMPVAL`：表示值为 0.01 的系数。
   d. 对 `h_data1` 的 16 个单精度赋值，依次为各元素的索引值；对 `h_weight` 的 16 个单精度赋值，依次为各元素索引值的平方，再乘以 `TMPVAL` 系数。
2. 主机端与设备端的数据传输：`with_sdaa` 是主机端的函数，被 `main` 函数调用。`with_sdaa` 调用 Kernel 函数，通过 SDAARuntime 接口将主机端的数据与 Device 端 Global 存储空间的数据进行传输。
   a. 设置计算核阵列 SPA 号：调用 `sdaaSetDevice` 接口指定 Kernel 函数运行在设备端具体哪一个计算核心阵列 SPA 上。
   b. 分配 Global 存储空间：通过 `sdaaMalloc` 接口在设备端分配 2 个 Global 存储空间 `（d_data1, d_data2）`。
   c. 从主机端内存向 Global 存储空间传输数据：通过 `sdaaMemcpy` 接口将主机端 `(h_data1, h_data2)` 分别拷贝到设备端 Global 存储空间 `（d_data1, d_data2）`。
   d. 调用 Kernel 函数：设置计算核心阵列 SPA 的个数，调用名为 `test` 的核函数，在设备端执行计算任务。
   e. 从 Global 存储空间拷贝计算结果到主机端内存：计算任务完成后，调用 `sdaaDeviceSynchronize` 接口同步设备，再通过 `sdaaMemcpy` 接口将设备端 Global 存储空间（`d_data1, d_data2`）的数据分别拷贝到 Host 端内存(`h_data1, h_data2`)。
   f. 释放 Global 存储空间：通过 `sdaaFree` 接口释放设备端的 Global 存储空间。

主机端与设备端的数据传输程序如下：

```cpp
/*
    * with_sdaa在设备端分配2个Global存储空间： d_data1, d_data2.
    * 将主机端h_data1, h_data2的数据拷贝至设备端Global存储空间
    * 进行向量操作和指数运算，再将计算完的数据从设备端拷贝回来。
    */
static void with_sdaa(float *h_data1, float *h_data2, int count, int size)
{
        // 指定Kernel函数运行在2号计算核心阵列SPA上
        sdaaSetDevice(2);
        float *d_data1 = NULL;
        float *d_data2 = NULL;

        // 在设备端的Global存储空间分配大小为size的连续内存，
        // 并将已分配内存的起始位置返回给指针d_data1.
        sdaaMalloc((void **)(&d_data1), size);

        // 将主机端内存h_data1的数据拷贝到设备端Global存储空间d_data1, 大小为size,
        // 方向是sdaaMemcpyHostToDevice。
        sdaaMemcpy(d_data1, h_data1, size, sdaaMemcpyHostToDevice);
        sdaaMalloc((void **)(&d_data2), size);
        sdaaMemcpy(d_data2, h_data2, size, sdaaMemcpyHostToDevice);

        // 设置计算核心阵列SPA的数量为1,调用名为test的核函数，在设备端执行计算任务。
        // 实参d_data1和d_data2是Device端Global空间上的地址，count是一个整型的变量。
        test<<<1>>>(d_data1, d_data2, count);

        // 同步设备
        sdaaDeviceSynchronize();

        // 将Global存储空间d_data1的数据拷贝到主机端h_data1，大小为size，
        // 方向是sdaaMemcpyDeviceToHost。
        sdaaMemcpy(h_data1, d_data1, size, sdaaMemcpyDeviceToHost);
        sdaaMemcpy(h_data2, d_data2, size, sdaaMemcpyDeviceToHost);

        sdaaFree(d_data1);   // 释放d_data1指针指向的Global存储空间
        sdaaFree(d_data2);   // 释放d_data2指针指向的Global存储空间
}
```

3. 打印运算结果：打印向量运算后的结果。`h_data1` 的数据是向量指数运算的结果，`h_data2` 的数据是向量乘加运算的结果。
4. 释放主机端内存：通过 `free` 接口释放主机端内存 `（h_data1, h_data2）`。

**主机端程序**

> 说明：
>
> Kernel 函数 `test` 和 Host 端的代码不在同一个 `.scpp` 文件时，需要在 Host 端代码所在的文件声明该 Kernel 函数，如：
>
> ```cpp
> extern __global__ void test(float *data1, float *data2, int number); // Kernel函数的声明
> ```

```cpp
 // Kernel函数的声明
extern __global__ void test(float *data1, float *data2, int number);


static void with_sdaa(float *h_data1, float *h_data2, int count, int size)
{
    // 指定Kernel函数运行在2号计算核心阵列SPA上
```

**设备端程序设计思路**

> 说明：
>
> 1. 使用 `__device__` 关键字修饰，说明该变量或函数是在设备端的 Global 存储空间，主机端不允许访问。
> 2. 使用 `__local__` 关键字修饰，说明该变量或函数是在设备端的 SPM 存储空间，主机端不允许访问。
> 3. 使用 `__global__` 关键字修饰，说明函数是 Kernel 函数，允许主机端调用，但不允许设备端调用。返回值类型仅为 void 类型。
> 4. `threadIdx` 用于读取计算核心 SPE 号。

1. 向量装载：通过 `simd_load` 接口将 64 字节长度的数据从连续内存区域中对齐装入到向量中。
2. 向量平方根运算：通过 `simd_sqrt` 接口计算向量平方根。
3. 向量指数运算：通过 `simd_exp` 接口对平方根运算的结果做向量指数运算。
4. 向量乘和向量加运算：通过 `*` 运算符做向量乘运算，`+` 运算符做向量加运算。
5. 向量小于等于比较：Kernel 函数调用设备端的 `test_vfcmp` 函数比较向量指数运算的结果是否小于等于向量乘加运算的结果。
   a. 向量小于等于比较操作：通过 `<=` 运算符做向量小于等于比较操作。逐元素比较两个向量，如果满足条件，就返回非 0 值（如：-1）；否则返回 0。
   b. 打印结果：打印比较结果。

### 15.2 使用 SPMD 完成矩阵乘

#### 15.2.1 概述

本例将在太初 AI 加速卡上通过 SPMD 编程模型实现矩阵乘操作。完整程序请参考完整程序。

主机端通过 SDAARuntime 接口分配设备端的 Global 存储空间，并将数据从主机端拷贝到设备端的 Global 存储空间。在设备端通过 SPMD 编程模型将计算任务按照可使用的计算核心 SPE 的数量进行切分。每个计算核心 SPE 通过 SDAA C 提供的数据搬运接口 `memcpy` 将所需的数据从 Global 存储空间搬运到 SPM 存储空间进行矩阵乘运算，最后将计算结果进行组合从而完成整个计算任务的求解。设备端完成计算后通过 SDAARuntime 接口将 Global 存储空间的计算结果拷贝到主机端内存。

#### 15.2.2 编程思路

**主机端**

1. 分配主机端内存并通过随机数进行初始化操作。
2. 主机端与设备端的数据传输。
   a. 设置计算核心阵列 SPA 号。
   b. 分配 Global 存储空间。
   c. 从主机端内存向 Global 存储空间传输数据。
   d. 调用 Kernel 函数。
   e. 将计算结果从 Global 存储空间传回主机端内存。
3. 验证计算结果的正确性。
4. 释放 Global 存储空间与主机端内存。

**设备端**

1. 计算每个计算核心 SPE 需要承担的计算任务。
2. 分配 SPM 存储空间。
3. 将计算数据从 Global 存储空间拷贝到 SPM 存储空间。
4. 执行矩阵乘操作。
5. 将计算结果从 SPM 存储空间拷贝到 Global 存储空间对应位置。
6. 释放 SPM 存储空间。

#### 15.2.3 编写程序

##### 15.2.3.1 主机端

**主机端程序设计思路**

1. 分配主机端内存并通过随机数进行初始化操作。
   a. 在主机端通过 `malloc` 接口分配 4 个主机端存储空间 `h_input`、`h_weight`、`h_output`、`h_output_reference`。
      - `h_input`：输入矩阵。
      - `h_weight`：权重矩阵。
      - `h_output`：输出矩阵。
      - `h_output_reference`：输出矩阵的参考结果。
   b. 通过随机数将输入矩阵与权重矩阵进行初始化操作。
   c. 在 Host 端模拟矩阵乘计算，并将计算结果存放到 `h_output_reference`。
2. 主机端与设备端的数据传输
   a. 设置计算核心阵列 SPA 号：通过 `sdaaSetDevice` 接口指定 Kernel 函数运行在哪一个计算核心阵列上。
   b. 分配 Global 存储空间：通过 `sdaaMalloc` 接口在设备端分配 3 个 Global 存储空间（`dev_input`、`dev_weight`、`dev_output`）。
      - `dev_input`：Global 存储空间内的输入矩阵。
      - `dev_weight`：Global 存储空间内的权重矩阵。
      - `dev_output`：Global 存储空间内的输出矩阵。
   c. 数据传输：通过 `sdaaMemcpy` 接口将 Host 端内存（`h_input`、`h_weight`、`h_output`）对应拷贝到 Device 端 Global 存储空间（`dev_input`、`dev_weight`、`dev_output`）。
   d. 调用 Kernel 函数：通过 `<<<…>>>` 调用 Kernel 函数。
   e. 拷贝计算结果：通过 `sdaaDeviceSynchronize` 接口等待计算核心 SPE 完成之前的所有请求任务，再通过 `sdaaMemcpy` 接口将 Device 端 Global 存储空间 `dev_output` 中的数据拷贝到 Host 内存 `h_output`。
   f. 计算结果对比：将 Kernel 的计算结果 `h_output` 与参考矩阵乘计算结果 `h_output_reference` 进行对比。
   g. 释放申请的 Device 端 Global 存储空间：通过 `sdaaFree` 接口释放 Device 端的 Global 存储空间。
   h. 释放申请的 Host 端存储空间：通过 `free` 接口释放申请的 Host 端存储空间。

**主机端程序**

```cpp
#include <time.h>


#define M 4096      // 输入矩阵和输出矩阵的行数
#define K 64        // 输入矩阵列数，也是权重矩阵行数
#define N 32             // 权重矩阵和输出矩阵的列数


int main()
{
    // 分配Host端空间
    // h_input为输入矩阵，h_weight为权重矩阵，h_output为计算结果矩阵
    // h_output_reference为Host模仿矩阵乘计算，计算结果存储矩阵
    short* h_input = (short*)malloc(M * K * sizeof(short));
    short* h_weight = (short*)malloc(K * N * sizeof(short));
    short* h_output = (short*)malloc(M * N * sizeof(short));
    short* h_output_reference = (short*)malloc(M * N * sizeof(short));

    srand(time(NULL));
    // 为Host端输入矩阵进行初始化
    for (int i = 0; i < M; i++) {
        for (int j = 0; j < K; j++) {
             h_input[i * K + j] = rand() % 10;
        }
    }

    // 为Host端权重矩阵进行初始化
    for (int i = 0; i < K; i++) {
        for (int j = 0; j < N; j++) {
             h_weight[i * N + j] = rand() % 5;
        }
    }

    // 在Host模拟矩阵乘操作，并将结果存放到h_output_reference
    memset(h_output_reference, 0, M * N * sizeof(short));
    for (int i = 0; i < M; i++) {
        for (int j = 0; j < N; j++) {
             for (int k = 0; k < K; k++) {
                 h_output_reference[i * N + j] +=
                     h_input[i * K + k] * h_weight[k * N + j];
             }
        }
    }

    // 配置使用0号计算核心阵列
    sdaaSetDevice(0);
    // 声明Device端地址
    // dev_input为输入矩阵，dev_weight为权重矩阵，dev_output为计算结果矩阵
    char* dev_input = NULL;
    char* dev_weight = NULL;
    char* dev_output = NULL;

    // 使用sdaaMalloc为Device端地址进行空间分配
    sdaaMalloc((void**)&dev_input, M * K * sizeof(short));
    sdaaMalloc((void**)&dev_weight, K * N * sizeof(short));
    sdaaMalloc((void**)&dev_output, M * N * sizeof(short));

    // 使用sdaaMemcpy将Host端内存拷贝到Device端
    sdaaMemcpy(dev_input, h_input, M * K * sizeof(short), sdaaMemcpyHostToDevice);
    sdaaMemcpy(dev_weight, h_weight, K * N * sizeof(short), sdaaMemcpyHostToDevice);

    // 启动核函数调用
    matmul_func<<<1>>>(dev_input, dev_weight, dev_output, M, K, N);

    // 等待计算核心SPE完成之前的所有请求任务
    sdaaDeviceSynchronize();

    // 使用sdaaMemcpy将Device端矩阵乘计算结果拷贝到Host端
    sdaaMemcpy(h_output, dev_output, M * N * sizeof(short), sdaaMemcpyDeviceToHost);

    // 将Device端矩阵乘计算结果与Host端计算结果进行对比
    // 验证失败则打印对应日志
    for (int i = 0; i < M * N; i++) {
             if(h_output[i] != h_output_reference[i]) {
             printf("find mismatching data on line %d: ace: %d, serial: %d\n",
                      i, (int)h_output[i], (int)h_output_reference[i]);
             break;
         }

         // 如果验证成功，则打印对应日志
         if (i == M * N - 1) {
             printf("matmul result compare ok\n");
         }
    }

    // 释放在Device端和Host申请的内存
    free(h_output_reference);
    free(h_output);
    free(h_weight);
    free(h_input);
    sdaaFree(dev_output);
    sdaaFree(dev_weight);
    sdaaFree(dev_input);

    return 0;
}
```

##### 15.2.3.2 设备端

**设备端程序设计思路**

设备端 Kernel 函数通过 SPMD 编程模型计算单个计算核心需要完成的计算子任务，每个计算核心 SPE 通过矩阵乘接口完成子任务的矩阵乘计算。最后将计算结果子集进行合并完成整体计算任务的求解。

> 说明：
>
> 1. 使用 `__device__` 关键字修饰，说明该变量或函数是在设备端的 Global 存储空间，主机端不允许访问。
> 2. 使用 `__global__` 关键字修饰，说明函数是 Kernel 函数，允许主机端调用，但不允许设备端调用。返回值类型仅为 void 类型。
> 3. `threadDim` 用于获取计算核心阵列可用的计算核心资源数量。
> 4. `threadIdx` 用于读取计算核心 SPE 的 ID。

1. 通过 SPMD 编程模型将计算任务进行拆分：通过 `threadDim` 计算每个计算核心 SPE 所承担的计算子任务。
2. 申请 SPM 存储空间：通过 `malloc` 申请 SPM 存储空间（`input`、`weight`、`output`）。
   a. `input`：输入矩阵。
   b. `weight`：权重矩阵。
   c. `output`：输出矩阵。
3. 从 Global 存储空间向 SPM 存储空间传输数据：通过非阻塞型 DMA 数据搬运接口将数据从 Global 存储空间拷贝到 SPM 存储空间。
4. 矩阵乘操作：通过非阻塞型矩阵乘接口完成子任务的矩阵乘计算。

> 说明：
>
> 调用矩阵乘接口需要 include 头文件 `#include <sdaa_matmul.h>`

5. 数据搬运：通过阻塞型 DMA 数据搬运接口 `memcpy` 将子任务的计算结果拷贝到 Global 存储空间的对应位置。
6. 释放 SPM 存储空间：通过 `free` 接口释放申请的 SPM 存储空间。

**设备端程序**

```cpp
#include <sdaa_matmul.h>
using namespace sdaa;


__device__ void matmul_compute_func(short *input, short *weight, short *output, int m, int k, int n)
{
     // 创建MatmulHandle以配置矩阵乘接口参数。此例中使用默认配置
     MatmulHandle handle;

     // 初始化矩阵乘接口，并配置输入和输出数据的类型
     matmul_init(handle, MatmulShortToShort);

     // 设置之后的计算不为最后一次计算，
     // 即计算结果会累加并缓存在矩阵乘计算单元的累加缓冲区内
     matmul_set_flushing_output(handle, false);

     // 将权重矩阵前32行加载到矩阵乘计算单元上
     matmul_load_weight(handle, weight, MatmulK32, MatmulN32);
     // 等待权重矩阵数据加载完成，此后可覆写weight上前32行数据
     matmul_wait_loading_weight(handle);

     // 计算加载输入矩阵的跨步步长，以64字节为单位
     const unsigned int stride = k / MatmulK32 - 1;

     // 跨步加载输入矩阵（此处相当于加载输入矩阵左半）
     // 并计算与权重矩阵的乘积
     matmul_compute(handle, input, m, MatmulK32, stride);
     // 等待输入矩阵加载完成，此后可覆写input上左半数据
     matmul_wait_loading_input(handle);

     // 使能下一次计算为最后一次计算
     // 计算完后会输出累加缓冲区内的数据
     matmul_set_flushing_output(handle, true);

     // 将权重矩阵后32行加载到矩阵乘计算单元上
     matmul_load_weight(handle, weight + MatmulK32 * MatmulN32, MatmulK32, MatmulN32);
     // 等待权重矩阵数据加载完成，此后可覆写weight上后32行数据
     matmul_wait_loading_weight(handle);

     // 跨步加载输入矩阵（此处相当于加载输入矩阵右半）
     // 并计算与权重矩阵的乘积
     matmul_compute(handle, input + MatmulK32, m, MatmulK32, stride);
     // 等待输入矩阵加载完成，此后可覆写input上右半数据
     matmul_wait_loading_input(handle);

     // 将矩阵乘运算结果写回到SPM指定位置
     matmul_store(handle, output, m, MatmulN32);

     // 等待运算结果写回完成
     matmul_wait(handle);
}


__global__ void matmul_func(char *data_input, char *data_weight, char *data_output, int m, int k, int n)
{
     // 计算每个计算核心SPE所需要承担的计算任务
     unsigned int slice = m / threadDim;

     // 申请SPM存储空间
     short *input = (short*)malloc(slice * k * sizeof(short));
     short *weight = (short*)malloc(k * n * sizeof(short));
     short *output = (short*)malloc(slice * n * sizeof(short));

     // 将数据从Global搬运输入矩阵和权重矩阵到SPM
     memcpy_async(input, data_input + threadIdx * slice * k * sizeof(short), slice * k * sizeof(short),
0, {0, 0}, MemcpyGlobalToSpm);
     memcpy_async(weight, data_weight, k * n * sizeof(short), 0, {0, 0}, MemcpyGlobalToSpm);
     // 等待异步拷贝完成
     memcpy_wait();

     // 矩阵乘计算
     matmul_compute_func(input, weight, output, slice, k, n);

     // 将输出矩阵从SPM搬运回Global
     memcpy(data_output + threadIdx * slice * n * sizeof(short), output, slice * n * sizeof(short));

     // 释放在SPM申请的内存
     free(output);
     free(weight);
     free(input);
}
```
