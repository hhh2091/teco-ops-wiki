## 13.3.2.2.1 非阻塞型 DMA 数据搬运（续：测试代码尾部与优化效果）

### 测试代码（续）

承接上一部分的非阻塞型 DMA 数据搬运示例，下面为测试代码主体函数的结尾部分。

```c
// init data
for (int i = 0; i < SIZE; i++) {
    data[i] = rand();
    for (int j = 0; j < TESTNUM; j++) {
        output[j][i] = 0;
    }
}

call_kernel(data, output);

for (int i = 0; i < TESTNUM; i++) {
    for (int j = 0; j < SIZE; j++) {
        if (fabs(output[i][j] - 1) > 1e-9) {
            errnum++;
            printf("data[%d] = %lf, output[%d][%d] = %lf\n", j, data[j], i,
                   j, output[i][j]);
        }
    }
}
if (!errnum) {
    printf("all pass!\n");
}
return 0;
}
```

在本示例中，使用非阻塞型 DMA 数据搬运接口进行数据搬运时，满足优化策略的性能相比最差情况：

- Global 向 SPM 存储空间进行数据搬运：约 2.47 倍。
- SPM 向 Global 存储空间进行数据搬运：约 3.29 倍。

## 13.3.2.2.2 阻塞型 DMA 数据搬运

### 13.3.2.2.2.1 概述

阻塞型 DMA 数据搬运通常比非阻塞型 DMA 数据搬运的使用限制少、通用性高，但性能会低于非阻塞型接口。

在数据搬运量相同的情况下，阻塞型 DMA 数据搬运可使用以下优化策略，性能会接近非阻塞型 DMA 数据搬运：

- 搬运地址
- 数据搬运量
- 数据搬运方向

### 13.3.2.2.2.2 搬运地址

本文主要介绍阻塞型 DMA 数据搬运地址的性能优化策略和示例。

#### 优化策略

阻塞型 DMA 数据搬运在数据搬运量相同、搬运方向相同的情况下：

- 当目的地址和源地址均满足 4B 对齐时，性能最好。
- 当目的地址和源地址的任意一个不满足 4B 对齐时，如果目标地址和源地址除以 4 所得的余数相同，性能较好。

#### 优化示例

单一线程使用阻塞型 DMA 数据搬运的方式，从 Global 到 SPM 存储空间搬运相同的数据量时，测试以下情况的执行周期并比较：

- 目的地址和源地址均满足 4B 对齐。
- 目的地址和源地址的任意一个不满足 4B 对齐，但目的地址和源地址除以 4 所得的余数相同。
- 目的地址和源地址的任意一个不满足 4B 对齐，但目的地址和源地址除以 4 所得的余数不同。

每个测试均调用 1000 次 memcpy 并计算平均值以消除初次调用时 cache 影响。

#### 测试环境

| TecoDriver 和 TecoToolKit 版本 | 计算核心频率 | 数据搬运量 |
|---|---|---|
| v1.2.0 | 2.36GHz | 128KB |

#### 测试代码

```cpp
#include <sdaa_perf.h>
using namespace sdaa;

#define SIZE 0x20000       // 128K

__device__ char global_arr[SIZE + 1];

__device__ void memcpy_perf(char *dst, char *src)
{
        // 打印内存拷贝的地址信息
        printf("dst addr: %p, src addr: %p\n", dst, src);
        // 数据初始化
        for (int i = 0; i < SIZE; i++) {
            dst[i] = 0;
            src[i] = 10;
        }
        // 采集完成拷贝128KB数据量的性能数据
        perf_start();
        for (int i = 0; i < 1000; i++) {
            memcpy(dst, src, SIZE * sizeof(char));
        }
        perf_stop();

        // 检查数据拷贝结果是否正确
        for (int i = 0; i < SIZE; i++) {
            if (dst[i] != 10) printf("memcpy results not correct!\n");
        }
}

// 地址为4B对齐场景下的数据拷贝
__global__ void test_aligned()
{
        if (threadIdx != 0) return;
        char *local_arr = (char *)malloc(SIZE * sizeof(char));
        memcpy_perf(local_arr, global_arr);
        free(local_arr);
        // 输出性能数据
        perf_print();
}

// 地址部分4B对齐场景下的数据拷贝
__global__ void test_partial_aligned()
{
    if (threadIdx != 0) return;
    char *local_arr = (char *)malloc((SIZE + 1) * sizeof(char));
    memcpy_perf(local_arr + 1, global_arr + 1);
    free(local_arr);
    // 输出性能数据
    perf_print();
}

// 地址不对齐场景下的数据拷贝
__global__ void test_unaligned()
{
    if (threadIdx != 0) return;
    char *local_arr = (char *)malloc((SIZE + 1) * sizeof(char));
    memcpy_perf(local_arr + 1, global_arr);
    free(local_arr);
    // 输出性能数据
    perf_print();
}

int main()
{
    // 设置使用2号SPA
    sdaaSetDevice(2);
    // kernel函数调用
    test_aligned<<<1>>>();
    // 等待kernel函数执行完成
    sdaaDeviceSynchronize();
    // kernel函数调用
    test_partial_aligned<<<1>>>();
    // 等待kernel函数执行完成
    sdaaDeviceSynchronize();
    // kernel函数调用
    test_unaligned<<<1>>>();
    // 等待kernel函数执行完成
    sdaaDeviceSynchronize();
    return 0;
}
```

#### 优化效果

| 地址对齐方式 | 平均每次 memcpy 的执行周期（拍） | 对应单线程的访存带宽 (GB/s) |
|---|---|---|
| 方式一：目的地址和源地址均满足 4B 对齐，例如：`dst addr: 0x1400, src addr: 0x6100040107e0` | 7362 | 45.45 |
| 方式二：目的地址和源地址的任意一个不满足 4B 对齐，但两者除以 4 所得的余数相同，例如：`dst addr: 0x1401, src addr: 0x6100040107e1` | 8704 | 35.06 |
| 方式三：目的地址和源地址的任意一个不满足 4B 对齐，且两者除以 4 所得的余数不同，例如：`dst addr: 0x1401, src addr: 0x6100040107e0` | 331260 | 0.92 |

在本示例中，阻塞型 DMA 数据搬运：

- 地址对齐方式一的访存带宽是方式三的约 49.40 倍。
- 地址对齐方式二的访存带宽是方式三的约 38.10 倍。

### 13.3.2.2.2.3 数据搬运量

本文主要介绍阻塞型 DMA 数据搬运量的性能优化策略和示例。

#### 优化策略

阻塞型 DMA 数据搬运在搬运地址对齐、搬运方向相同的情况下，在满足以下条件时，性能较好：

数据搬运量 4B 对齐，即数据搬运量的大小是 4B 的整数倍。

#### 优化示例

单一线程使用阻塞型 DMA 数据搬运的方式，从 Global 到 SPM 存储空间的相同 4B 对齐地址搬运数据时，测试以下情况的执行周期并比较：

- 数据搬运量为 128KB，是 4B 的整数倍。
- 数据搬运量为（128KB - 1B），不是 4B 的整数倍。
- 数据搬运量为（128KB - 2B），不是 4B 的整数倍。
- 数据搬运量为（128KB - 3B），不是 4B 的整数倍。
- 数据搬运量为（128KB - 4B），是 4B 的整数倍。

每个测试均调用 1000 次 memcpy 并计算平均值以消除初次调用时 cache 影响。

#### 测试环境

| TecoDriver 和 TecoToolKit 版本 | 计算核心频率 |
|---|---|
| v1.2.0 | 2.36GHz |

### 13.3.2.2.2.4 数据搬运方向

本文主要介绍阻塞型 DMA 数据搬运方向的性能优化策略和示例。

#### 优化策略

阻塞型 DMA 数据搬运在数据搬运量相同、搬运地址对齐的情况下，在不同的搬运方向下，性能由高到低排列如下：

1. 同一 SPM 到 SPM 存储空间
2. Global 到 SPM 存储空间
3. SPM 到 Global 存储空间
4. Global 到 Global 存储空间

#### 优化示例

单一线程使用阻塞型 DMA 数据搬运的方式，在搬运数据量相同、搬运地址对齐时，测试以下不同数据搬运方向的执行周期并比较：

1. 同一 SPM 到 SPM 存储空间
2. Global 到 SPM 存储空间
3. SPM 到 Global 存储空间
4. Global 到 Global 存储空间

每个测试均调用 1000 次 memcpy 并计算平均值以消除初次调用时 cache 影响。

#### 测试环境

| TecoDriver 和 TecoToolKit 版本 | 计算核心频率 | 数据搬运量 |

| v1.2.0 | 2.36GHz | 96KB |

## 13.3.2.2.3 非阻塞型 DMA 数据搬运

本文主要介绍非阻塞型 DMA 数据搬运的性能优化策略和示例。

### 优化策略

如果搬运的数据与当前程序的后续操作不存在数据依赖时，使用非阻塞型接口可以更好的进行流水排布，相对阻塞型接口性能更好。

### 优化示例

在设备端测试以下情况的数据计算总执行周期并比较：

- 使用阻塞型数据搬运接口：先执行数据搬运，再执行与搬运数据无依赖的计算操作。
- 使用非阻塞型数据搬运接口：同步执行数据搬运和与搬运数据无依赖的计算操作。

每个测试均调用 10000 次数据搬运和与搬运数据无依赖的计算操作，从而计算平均值以消除初次调用时 cache 影响。

### 测试环境

| TecoDriver 和 TecoToolKit 版本 | 计算核心频率 | 数据搬运量 |
|---|---|---|
| v1.2.0 | 2.36GHz | 10KB |

### 测试代码

```cpp
#include <sdaa_perf.h>
using namespace sdaa;

#define SIZE 2560
#define SIZE_CAL 256
#define TIME 10000

__device__ void test_async(int *d_dst, int *d_cal)
{
    int *spm_src = (int *)malloc(SIZE * sizeof(int));
    for (int i = 0; i < SIZE; i++) {
        spm_src[i] = i;
    }
    // 统计性能数据
    perf_start();

    intv16 cal_src;
    intv16 add = 1;

    for (int time = 0; time < TIME; time++) {
        // 发起非阻塞式数据搬运
        memcpy_async(d_dst, spm_src, SIZE * sizeof(int), 0, {0, 0}, MemcpySpmToGlobal);

        // 同步进行与数据搬运无数据依赖的计算
        for (int idx = 0; idx < SIZE_CAL; idx += 16) {
            // 使用向量进行计算
            simd_load(cal_src, d_cal + idx);
            intv16 tmp = cal_src + add;
            simd_store(tmp, d_cal + idx);
        }

        // 等待完成非阻塞式数据搬运
        memcpy_wait();
    }
    // 统计性能数据
    perf_stop();
    free(spm_src);
}

__device__ void test_sync(int *d_dst, int *d_cal)
{
    int *spm_src = (int *)malloc(SIZE * sizeof(int));
    for (int i = 0; i < SIZE; i++) {
        spm_src[i] = i;
    }
    // 统计性能数据
    perf_start();

    intv16 cal_src;
    intv16 add = 1;

    for (int time = 0; time < TIME; time++) {
        // 发起阻塞式数据搬运
        memcpy(d_dst, spm_src, SIZE * sizeof(int));

        // 异步进行与数据搬运无数据依赖的计算
        for (int idx = 0; idx < SIZE_CAL; idx += 16) {
            // 使用向量进行计算
            simd_load(cal_src, d_cal + idx);
            intv16 tmp = cal_src + add;
```

## 13.3.2.3 RMA 数据搬运

### 13.3.2.3.1 SPE ID 相关

本文主要介绍 RMA 数据搬运的性能优化策略和示例。

#### 优化策略

进行 RMA 数据搬运的两个 SPE 的 ID 号 x 和 y，在满足以下条件时，性能较好：

令 D(x, y) = abs(x/8 - y/8) + abs(x%8 - y%8)，则 D(x, y) 的值越小，性能越好。

例如：SPE0 和 SPE9 的 RMA 数据搬运性能要优于 SPE0 和 SPE31 的 RMA 数据搬运性能。

#### 优化示例

测试并比较以下两种场景的阻塞型 RMA 数据搬运的执行周期：

- SPE0 和 SPE9
- SPE0 和 SPE31

每个测试均调用 1000 次 rma_put 并计算平均值以消除初次调用时 cache 影响。

#### 测试环境

| TecoDriver 和 TecoToolKit 版本 | 计算核心频率 |
|---|---|
| v1.2.0 | 2.36GHz |

#### 测试代码

```cpp

__local__ int dst;

__device__ void rma_perf(unsigned long target_id)

        if (threadIdx != 0 && threadIdx != target_id) return;
        // data init
        src = 0;

        if (threadIdx == 0) src = 100;
```

### 13.3.2.3.2 非阻塞型 RMA 数据搬运

#### 13.3.2.3.2.1 优化策略

1. 如果搬运的数据与当前程序的后续操作不存在数据依赖时，使用非阻塞型接口可以更好的进行流水排布，相对阻塞型接口性能更好。
2. 当使用非阻塞型接口进行数据搬运时，使用 RmaCustomizeMode 模式相对于 RmaNormalMode 模式，性能更好。

#### 13.3.2.3.2.2 rma_get 优化示例

- 使用非阻塞型数据搬运接口：使用 rma_async_get 的 RmaNormalMode 模式，同步执行数据搬运和与搬运数据无依赖的计算操作。

## 13.3.2.4 Broadcast 数据搬运

### 13.3.2.4.1 SPE ID 相关

本文主要介绍使用线程组进行 Broadcast 数据搬运的性能优化策略和示例。

#### 优化策略

使用线程组进行 Broadcast 数据搬运时，当线程组内的所有 SPE 满足以下任意一种条件时，性能较好：

- 线程组内所有 SPE 刚好能被划分为：{0~7}，{8~15}，{16~23}，{24~31} 中的一组或多组。
- 线程组内所有 SPE 刚好能被划分为：{0, 8, 16, 24}，{1, 9, 17, 25}，{2, 10, 18, 26}，{3, 11, 19, 27}，{4, 12, 20, 28}，{5, 13, 21, 29}，{6, 14, 22, 30}，{7, 15, 23, 31} 中的一组或多组。

例如：对包含 8 个 SPE 的线程组进行广播时，线程组 A={0, 1, 2, 3, 4, 5, 6, 7} 或 B={0, 1, 8, 9, 16, 17, 24, 25} 的性能要优于线程组 C={0, 1, 9, 10, 18, 19, 27, 28}。

#### 优化示例

对使用线程组 A={0, 1, 2, 3, 4, 5, 6, 7}、B={0, 1, 8, 9, 16, 17, 24, 25} 和 C={0, 1, 9, 10, 18, 19, 27, 28} 进行阻塞型广播的（本部分内容在第 g02 部分行范围末尾被截断，后续内容见下一部分）。
