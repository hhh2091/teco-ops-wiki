## 3. 性能度量

### 3.1 性能指标

本文主要介绍衡量程序性能的关键指标：执行周期和访存带宽。

#### 执行周期

在SDAA C中，使用执行周期作为衡量程序性能的关键指标。

程序性能公式如下：

```
性能 = (指令 / 程序) · (执行周期 / 指令) · (时间 / 执行周期)
     = 1 / (程序 · 执行周期 · 时钟周期)
```

其中：

- **时钟周期（单位：秒）**：计算核心SPE基本的时间单位，指发生一个最小事件所需的时间，由计算核心SPE的时钟频率决定。

  时钟周期的公式：

  ```
  时钟周期 = 1 / (SPE 的时钟频率)
  ```

- **执行周期（单位：拍）**：程序运行所消耗的时钟周期个数（也称作拍数，一拍对应一个时钟周期）。

由于时钟周期是由计算核心SPE的时钟频率所决定的固定值，那么执行周期越短，程序的运行效率越高效，程序性能越好。因此，执行周期是衡量程序性能的关键指标。

> **说明：**
> - 非访存类指令的执行周期通常是固定的。
> - 访存类指令的执行周期与数据量、访存方式以及时钟频率有关。

#### 访存带宽

对于以数据读写为主的程序，访存带宽（即IO带宽）是通用的性能指标。

访存带宽是计算核心SPE在单位时间内读取或写入内存的数据量，通常以每秒传输的比特数或字节数进行衡量。

##### 理论带宽

理论带宽的计算公式如下：

```
理论带宽（Byte/s） = 内存最大频率 × 内存通道数 × 648（每次传输位宽，单位：bit）
```

##### 访存带宽

在实际使用中，由于算法效率、内存延迟等均会影响数据传输的效率，因此访存带宽通常低于理论带宽。

访存带宽的计算公式如下：

```
访存带宽（Byte/s） = 数据传输量（Byte） × 频率（Hz） / 执行周期（拍）
```

由以上公式可以看出：执行周期越短，访存带宽越大。

### 3.2 性能采样

使用 `perf_start`、`perf_stop` 和 `perf_print` 函数可以对SDAA C程序进行性能采样，采集目标代码段的执行周期。目前，性能采样工具只输出两个主要的性能数据，一个是 `perf_start`、`perf_stop` 之间的执行周期（延时拍数），另一个是指令缓存脱靶次数（Instruction Cache Miss）。

> **说明：**
> 使用性能采样函数前，需要手动包含 `sdaa_perf.h` 头文件。

- **perf_start**（详情请参考《SDAA C编程指南》的 perf_start 章节）：开始性能采样接口，通过此接口标识性能采样的起始位置。
- **perf_stop**（详情请参考《SDAA C编程指南》的 perf_stop 章节）：结束性能采样接口，通过此接口标识性能采样的结束位置。
- **perf_print**（详情请参考《SDAA C编程指南》的 perf_print 章节）：在终端界面输出采集到的性能数据（延时拍数和指令缓存脱靶次数）。

#### 性能采样示例

> **说明：**
> 为准确测量性能，可以循环多次调用被测程序并计算执行周期的平均值，以减少缓存未命中（Cache Miss）所带来的性能偏差。

使用 `perf_start` 和 `perf_stop` 进行性能采样的示例如下：

```cpp
#include <sdaa_perf.h>
using namespace sdaa;


#define NUM 51200
#define FSIZE (NUM * sizeof(float))


__global__ void kernel_sig(float *a, uint32_t num)
{
    if (threadIdx == 0){
        perf_start();
        for (uint32_t i = 0; i < num; i++) {
            a[i] = sigmoidf(a[i]);
        }
        perf_stop();
        perf_print();
    }
}


int main()
{
    sdaaSetDevice(0);
    // 主机端申请数据空间
    float *host_data = (float *)malloc(FSIZE);
    for (int i = 0; i < NUM; i++) {
         host_data[i] = (float)i / NUM;
    }
    float *dev_input = NULL;
    // 申请Device端数据空间
    sdaaMalloc((void **)&dev_input, FSIZE);
    // 将主机端数据复制到Device端
    sdaaMemcpy(dev_input, host_data, FSIZE, sdaaMemcpyHostToDevice);
    // 核函数运行计算
    kernel_sig<<<1>>>(dev_input, NUM);
    sdaaDeviceSynchronize();
    // 将计算好的数据从Device端复制到主机端
    sdaaMemcpy(host_data, dev_input, FSIZE, sdaaMemcpyDeviceToHost);


    free(host_data);
    sdaaFree(dev_input);
    return 0;
}
```

#### 性能采样结果

完成性能测试后，调用 `perf_print` 函数打印出性能采样结果。性能采样结果将提供关于代码性能的详细信息：

```
Kernel perf data of thread 0:
 --------------------------------------------------------------------------------------
|START LINE NUM                       |10                                              |
|LATENCY                              |35004840                                        |
|CNT_L1IC_MISS                        |51205                                           |
```

#### 输出说明

| 字段 | 说明 |
|------|------|
| START LINE NUM | 测试开始的行号。 |
| LATENCY | 延迟时间，即执行周期，单位：拍。**说明：** 具体结果可能随着不同的硬件上下波动。 |
| CNT_L1IC_MISS | 指令缓存脱靶次数。 |

## 4. 性能优化

### 4.1 程序并行

#### 4.1.1 任务级并行

本文主要介绍任务级并行的性能优化策略和示例。

##### 优化策略

太初AI加速卡目前提供多个计算核心阵列SPA，每个SPA构成一个独立的进程。任务级并行主要指：同时并行运行多个计算核心阵列SPA完成相应的计算任务。

##### 优化示例

计算40000个数据的sigmoid输出值。

###### 测试环境

| TecoDriver和TecoToolKit版本 | 计算核心频率 |
|----------------------------|-------------|
| v1.2.0 | 2.36GHz |

###### 优化前

只使用一个SPA计算40000个数的sigmoid输出值。

```cpp
#include <sdaa_perf.h>
using namespace sdaa;


#define DEV_NUM 4
#define NUM 10000
#define FSIZE (NUM * sizeof(float))


__global__ void kernel_sig(float *a, uint32_t num)
{
        if (threadIdx == 0){
            perf_start();
            for (uint32_t i = 0; i < num; i++) {
                *(a + i) = sigmoidf(*(a + i));
            }
            perf_stop();
            perf_print();
        }
}


int main()
{
    sdaaSetDevice(0);
    // 主机端申请数据空间
    float *host_data = (float *)malloc(DEV_NUM * FSIZE);
    for (int i = 0; i < (DEV_NUM * NUM); i++) {
        host_data[i] = (float)i / NUM;
    }
    float *dev_input = NULL;
    // 申请Device端数据空间
    sdaaMalloc((void **)&dev_input, (DEV_NUM * FSIZE));
    // 将主机端数据复制到Device端
    sdaaMemcpy(dev_input, host_data, (DEV_NUM * FSIZE), sdaaMemcpyHostToDevice);
    // 核函数运行计算
    kernel_sig<<<1>>>(dev_input, (DEV_NUM * NUM));
    sdaaDeviceSynchronize();
    // 将计算好的数据从Device端复制到主机端
    sdaaMemcpy(host_data, dev_input, (DEV_NUM * FSIZE), sdaaMemcpyDeviceToHost);


    free(host_data);
    sdaaFree(dev_input);
    return 0;
}
```

###### 优化后

将40000个数改为进程间共享数据，本文以4等份为例，同时并行运行4个SPA计算40000个数的sigmoid输出值，每个SPA计算一份数据。

```cpp
#include <sdaa_perf.h>
#include <fcntl.h>
#include <iostream>
#include <string>
#include <sys/mman.h>
#include <sys/types.h>
#include <sys/wait.h>
#include <unistd.h>
using namespace sdaa;


#define DEV_NUM 4
#define NUM 10000
#define FSIZE (NUM * sizeof(float))


__global__ void kernel_sig(float *a, uint32_t num)
{
    if(threadIdx == 0){
        perf_start();
        for (uint32_t i = 0; i < num; i++) {
             *(a + i) = sigmoidf(*(a + i));
        }
        perf_stop();
        perf_print();
    }
}


int main()
{
    float *shared_mem;


    // 创建共享内存
    int fd = shm_open("/my_shared_memory", O_CREAT | O_RDWR, S_IRUSR | S_IWUSR);
    ftruncate(fd, (DEV_NUM * FSIZE));


    // 共享内存 大小分成 4 份
    shared_mem = (float *)mmap(NULL, (DEV_NUM * FSIZE), PROT_READ | PROT_WRITE,
                               MAP_SHARED, fd, 0);


    for (int i = 0; i < (DEV_NUM * NUM); i++) {
        shared_mem[i] = (float)i / NUM;
    }
    // 创建第一个子进程
    pid_t pid1 = fork();
    if (pid1 == 0) {
        // 第一个子进程中使用第一个计算核心阵列SPA
        sdaaSetDevice(0);
        // 在Device端申请内存，大小是 共享内存的 1/4
        float *dev_input = NULL;
        sdaaMalloc((void **)&dev_input, FSIZE);
        // 将共享内存的第一个1/4数据复制到 第一个device 的内存上
        sdaaMemcpy(dev_input, shared_mem, FSIZE, sdaaMemcpyHostToDevice);
        // 运行kernel 函数 进行计算
        kernel_sig<<<1>>>(dev_input, NUM);
        sdaaDeviceSynchronize();
        // 将计算过的 device 上的数据复制到第一个1/4的共享内存
        sdaaMemcpy(shared_mem, dev_input, FSIZE, sdaaMemcpyDeviceToHost);
        sdaaFree(dev_input);
        exit(0);
    }


    // 创建第二个子进程
    pid_t pid2 = fork();
    if (pid2 == 0) {
        // 第二个子进程中使用第二个计算核心阵列SPA
        sdaaSetDevice(1);
        // 在Device端申请内存，大小是 共享内存的 1/4
        float *dev_input = NULL;
        sdaaMalloc((void **)&dev_input, FSIZE);
        // 将共享内存的第二个1/4数据复制到 第二个device 的内存上
        sdaaMemcpy(dev_input, (shared_mem + NUM), FSIZE,
                   sdaaMemcpyHostToDevice);
        // 运行kernel 函数 进行计算
        kernel_sig<<<1>>>(dev_input, NUM);
        sdaaDeviceSynchronize();
        // 将计算过的 device 上的数据复制到第二个1/4的共享内存
        sdaaMemcpy((shared_mem + NUM), dev_input, FSIZE,
                   sdaaMemcpyDeviceToHost);
        sdaaFree(dev_input);
        exit(0);
    }


    // 创建第三个子进程
    pid_t pid3 = fork();
    if (pid3 == 0) {
        // 第三个子进程中使用第三个计算核心阵列SPA
        sdaaSetDevice(2);
        // 在Device端申请内存，大小是 共享内存的 1/4
        float *dev_input = NULL;
        sdaaMalloc((void **)&dev_input, FSIZE);
        // 将共享内存的第三个1/4数据复制到 第三个device 的内存上
        sdaaMemcpy(dev_input, (shared_mem + 2 * NUM), FSIZE,
                   sdaaMemcpyHostToDevice);
        // 运行kernel 函数 进行计算
        kernel_sig<<<1>>>(dev_input, NUM);
        sdaaDeviceSynchronize();
        // 将计算过的 device 上的数据复制到第三个1/4的共享内存
        sdaaMemcpy((shared_mem + 2 * NUM), dev_input, FSIZE,
                   sdaaMemcpyDeviceToHost);
        sdaaFree(dev_input);
        exit(0);
    }


    // 创建第四个子进程
    pid_t pid4 = fork();
    if (pid4 == 0) {
        // 第四个子进程中使用第四个计算核心阵列SPA
        sdaaSetDevice(3);
        // 在Device端申请内存，大小是 共享内存的 1/4
        float *dev_input = NULL;
        sdaaMalloc((void **)&dev_input, FSIZE);
        // 将共享内存的第四个1/4数据复制到 第四个device 的内存上
        sdaaMemcpy(dev_input, (shared_mem + 3 * NUM), FSIZE,
                   sdaaMemcpyHostToDevice);
        // 运行kernel 函数 进行计算
        kernel_sig<<<1>>>(dev_input, NUM);
        sdaaDeviceSynchronize();
        // 将计算过的 device 上的数据复制到第四个1/4的共享内存
        sdaaMemcpy((shared_mem + 3 * NUM), dev_input, FSIZE,
                   sdaaMemcpyDeviceToHost);
        sdaaFree(dev_input);
        exit(0);
    }


    pid_t pid;
    int status = 0;
    // 等待子进程全部执行完成
    while ((pid = wait(&status)) > 0)
        ;


    // remove share memory
    munmap(shared_mem, (4 * FSIZE));
    shm_unlink("/my_shared_memory");


    return 0;
}
```

#### 4.1.2 线程级并行（SPMD）

本文主要介绍线程级并行（SPMD）的性能优化策略和示例。

##### 优化策略

太初AI加速卡目前提供多个计算核心SPE，每个SPE构成一个独立的线程。线程级并行（SPMD（详情请参考《SDAA C编程指南》的 SPMD 章节））主要指：每个计算核心SPE并行完成不同数据的相同任务，从而发挥设备的并行处理能力。

> 图1 线程级并行（SPMD）

如上图，使用SPMD优化前是所有线程操作同一个数据，优化后是每个线程操作不同的数据。因此，线程级并行可以大幅度提高数据处理的性能。

##### 优化示例

计算51200个数据的sigmoid输出值。

###### 测试环境

| TecoDriver和TecoToolKit版本 | 计算核心频率 |
|----------------------------|-------------|
| v1.2.0 | 2.36GHz |

###### 优化后

通过 threadIdx（详情请参考《SDAA C编程指南》的 threadIdx 章节）给每个SPE（线程）分配不同的数据，从而实现 threadDim（详情请参考《SDAA C编程指南》的 threadDim 章节）个数据的并行计算。

```cpp
#include <sdaa_perf.h>
using namespace sdaa;


#define NUM 51200
#define FSIZE (NUM * sizeof(float))


__global__ void kernel_sig(float *a, uint32_t num)
{
    perf_start();
    // 每次循环计算是跨步为 threadDim，为每个SPE分配一个数据去计算
    for (uint32_t i = 0; i < num; i += threadDim) {
        // 用 i+threadIdx 作为坐标，每个SPE都参与计算
        a[i + threadIdx] = sigmoidf(a[i + threadIdx]);
    }
    perf_stop();
    perf_print();
}


int main()
{
    sdaaSetDevice(0);
    // 主机端申请数据空间
    float *host_data = (float *)malloc(FSIZE);
    for (int i = 0; i < NUM; i++) {
        host_data[i] = (float)i / NUM;
    }
    float *dev_input = NULL;
    // 申请Device端数据空间
    sdaaMalloc((void **)&dev_input, FSIZE);
    // 将主机端数据复制到Device端
    sdaaMemcpy(dev_input, host_data, FSIZE, sdaaMemcpyHostToDevice);
    // 核函数运行计算
```

#### 4.1.3 指令级并行（SIMD）

本文主要介绍指令级并行（SIMD）的性能优化策略和示例。

##### 优化策略

太初AI加速卡的每个计算核心SPE都有多个向量寄存器，运用SIMD（Single Instruction Multiple Data，单指令流多数据流）向量指令的加速能力，不仅可以降低功耗，还可以提高性能，满足系统需要。

##### 优化示例

###### 优化前

使用SPMD编程模型，一个SPA内所有SPE分别执行1600次浮点标量计算，实现51200次浮点加法。

```cpp
#include <sdaa_perf.h>
using namespace sdaa;


#define NUM 51200
#define FSIZE (NUM * sizeof(float))


__global__ void kernel_add(float *a, uint32_t num)
{
        perf_start();

            a[i + threadIdx] = a[i + threadIdx] + 10;
        }
        perf_stop();
        perf_print();
}

{
```

###### 优化后

使用SPMD编程模型和SIMD优化，一个SPA内所有SPE分别执行100次浮点向量计算，实现51200次浮点加法。

```cpp

#define NUM 51200
#define FSIZE (NUM * sizeof(float))

        // 将 每个SPE的向量寄存器里的16个float数据， 存回到对应地址
        simd_store(va, (a + i + threadIdx * SIMD_NUM));
    }
    perf_stop();
    perf_print();
}

{

        host_data[i] = (float)i / NUM;

    float __attribute__((aligned(64))) *dev_input = NULL;
    // 申请Device端数据空间
    sdaaMalloc((void **)&dev_input, FSIZE);
    // 将主机端数据复制到Device端
    sdaaMemcpy(dev_input, host_data, FSIZE, sdaaMemcpyHostToDevice);
    // 核函数运行计算
    kernel_add<<<1>>>(dev_input, NUM);
    sdaaDeviceSynchronize();
    // 将计算好的数据从Device端复制到主机端
    sdaaMemcpy(host_data, dev_input, FSIZE, sdaaMemcpyDeviceToHost);


    free(host_data);
    sdaaFree(dev_input);
    return 0;
}
```

#### 4.2.2 DMA数据搬运

##### 4.2.2.1 SPE ID相关

本文主要介绍使用不同线程组进行DMA数据搬运的性能优化策略和示例。

###### 优化策略

使用DMA数据搬运接口时，在数据搬运量一定的情况下，加入线程组的SPE满足以下条件时，性能较好：

- SPE的ID号除以8所得的余数尽可能不相同。
- 在无法避免需使用有相同余数的SPE情况下，启用的有相同余数的SPE的数量尽可能小。

例如：使用8个SPE的线程组进行数据搬运时：

- 线程组 A={0, 1, 2, 3, 4, 5, 6, 7} 的性能要优于线程组 B={0, 1, 2, 3, 8, 9, 10, 11}。
- 线程组 B={0, 1, 2, 3, 8, 9, 10, 11} 的性能要优于线程组 C={0, 1, 8, 9, 16, 17, 24, 25}。

> **提示：**
> 该优化策略适用于非阻塞型DMA数据搬运及阻塞型DMA数据搬运中非SPM存储空间之间的数据搬运。

###### 优化示例

使用8个SPE的线程组进行非阻塞型DMA数据搬运时，测试以下几种线程组的执行周期并比较：

- A={0, 1, 2, 3, 4, 5, 6, 7}，余数均不相同，即相同余数的SPE数量最大值为1。
- B={0, 1, 2, 3, 8, 9, 10, 11}，有2组相同的余数（0, 1, 2, 3），相同余数的SPE数量最大值为2。
- C={0, 1, 8, 9, 16, 17, 24, 25}，有4组相同的余数（0, 1），相同余数的SPE数量最大值为4。

##### 4.2.2.2 阻塞型DMA数据搬运

###### 4.2.2.2.1 概述

阻塞型DMA数据搬运通常比非阻塞型DMA数据搬运的使用限制少、通用性高，但性能会低于非阻塞型接口。

在数据搬运量相同的情况下，阻塞型DMA数据搬运可使用以下优化策略，性能会接近非阻塞型DMA数据搬运：

- 搬运地址
- 数据搬运量
- 数据搬运方向

###### 4.2.2.2.2 搬运地址

本文主要介绍阻塞型DMA数据搬运地址的性能优化策略和示例。

**优化策略**

阻塞型DMA数据搬运在数据搬运量相同、搬运方向相同的情况下：

- 当目的地址和源地址均满足4B对齐时，性能最好。
- 当目的地址和源地址的任意一个不满足4B对齐时，如果目标地址和源地址除以4所得的余数相同，性能较好。

**优化示例**

- 目的地址和源地址均满足4B对齐。
- 目的地址和源地址的任意一个不满足4B对齐，但目的地址和源地址除以4所得的余数相同。
- 目的地址和源地址的任意一个不满足4B对齐，但目的地址和源地址除以4所得的余数不同。

###### 优化策略

- 使用阻塞型数据搬运接口：先执行数据搬运，再执行与搬运数据无依赖的计算操作。
- 使用非阻塞型数据搬运接口：同步执行数据搬运和与搬运数据无依赖的计算操作。

###### 测试环境

```cpp

using namespace sdaa;


#define SIZE 2560
#define SIZE_CAL 256
#define TIME 10000


__device__ void test_async(int *d_dst, int *d_cal)
{
        int *spm_src = (int *)malloc(SIZE * sizeof(int));
        for (int i = 0; i < SIZE; i++) {
            spm_src[i] = i;
        }
        // 统计性能数据
        perf_start();

        intv16 add = 1;

            // 发起非阻塞式数据搬运

            // 同步进行与数据搬运无数据依赖的计算
            for (int idx = 0; idx < SIZE_CAL; idx += 16) {
                // 使用向量进行计算
                simd_load(cal_src, d_cal + idx);
                intv16 tmp = cal_src + add;
                simd_store(tmp, d_cal + idx);
            }


            // 等待完成非阻塞式数据搬运
            memcpy_wait();
        }
        // 统计性能数据
        perf_stop();

}

        }

{
    if (threadIdx != 0) {
        return;
    }


    test_async(d_dst_async, d_cal_async);
    test_sync(d_dst_sync, d_cal_sync);

    int *d_cal_sync = NULL;

    for (int i = 0; i < SIZE; i++) {
```

#### 4.2.3 RMA数据搬运

##### 4.2.3.1 SPE ID相关

本文主要介绍RMA数据搬运的性能优化策略和示例。

###### 优化策略

进行RMA数据搬运的两个SPE的ID号 x 和 y，在满足以下条件时，性能较好：

令 D(x, y) = abs(x/8 - y/8) + abs(x%8 - y%8)，则 D(x, y) 的值越小，性能越好。

例如：SPE0 和 SPE9 的RMA数据搬运性能要优于 SPE0 和 SPE31 的RMA数据搬运性能。

#### 4.2.4 Broadcast数据搬运

##### 4.2.4.1 SPE ID相关

本文主要介绍使用线程组进行Broadcast数据搬运的性能优化策略和示例。

###### 优化策略

使用线程组进行Broadcast数据搬运时，当线程组内的所有SPE满足以下任意一种条件时，性能较好：

- 线程组内所有SPE刚好能被划分为：{0~7}，{8~15}，{16~23}，{24~31} 中的一组或多组。
- 线程组内所有SPE刚好能被划分为：{0, 8, 16, 24}，{1, 9, 17, 25}，{2, 10, 18, 26}，{3, 11, 19, 27}，{4, 12, 20, 28}，{5, 13, 21, 29}，{6, 14, 22, 30}，{7, 15, 23, 31} 中的一组或多组。

例如：对包含8个SPE的线程组进行广播时，线程组 A={0, 1, 2, 3, 4, 5, 6, 7} 或 B={0, 1, 8, 9, 16, 17, 24, 25} 的性能要优于线程组 C={0, 1, 9, 10, 18, 19, 27, 28}。

###### 优化示例
