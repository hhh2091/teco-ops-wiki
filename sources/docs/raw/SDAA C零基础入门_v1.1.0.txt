1. 概述
1.1 什么是算子开发
算子（Operator）通常是指在深度学习、机器学习或其他数值计算领域中，用于执行特定数学操作的函数，例如：加
法、矩阵乘、卷积等。

算子是模型训练推理的重要组成部分，主要用于构建、部署深度学习神经网络模型等。一个高性能的算子可以提升模
型的性能，加速模型的训练和推理。

在以下场景，需要进行算子开发：

   训练场景下，将模型迁移到太初AI加速卡上，存在不支持的算子。

   推理场景下，将模型转换为太初推理框架支持的模型时，存在不支持的算子。

   提升模型性能，需要开发更高性能的算子。


1.2 算子基本概念
 算子类型（Type）

算子类型用于描述实现逻辑相同的一类算子，例如：卷积类算子、池化类算子等。


     说明：

         在一个神经网络中，同一类型的算子可能会存在多个，例如：在某个神经网络中，存在2个卷积类算子
         conv1，conv2。

         同一类型算子的实现可能会存在差别，例如：conv1，conv2都属于卷积类算子，但实现会根据具体的使
         用情况而不同。



 算子名称（Name）

神经网络中具体算子的名称，算子名称在同一神经网络中需要保持唯一。

 算子数据结构（Tensor）

张量（Tensor）是存储算子输入数据和输出数据的基本数据结构。张量的属性主要包括：名称（name）、形状
（shape）、数据类型（dtype）和数据排布格式（format）。

 张量名称（name）

张量的名称，可以根据名称对张量进行索引。

 张量的数据类型（dtype）

张量中存储的数据的类型，取值范围：int8、int16、float16、float32等。

 张量的数据排布格式（format）

在内存中张量数据的存储格式，不同的数据排布格式会影响数据的访问模式、内存利用率和算子的性能。

以四维张量(N, H, W, C)为例，四个维度结合图像方面的物理含义如下：

   N（Batch）：一个批次内图片的数量，即一次处理的图片数量。

   H（Height）：特征图的高，即垂直高度方向的像素个数。

   W（Width）：特征图的宽，即水平宽度方向的像素个数。

   C（Channels）：特征图的通道数，例如：灰度图像为通道数1，彩色RGB图像为3。

1.3 SDAA编程模型
1.3.1 SDAA异构并行计算平台
SDAA C（Software Defined Accelerator Architecture C）是运行在SDAA异构并行计算平台上的编程模型。以太
初AI加速卡为计算核心，支持C/C++编程语言，您仅需学习少量的SDAA C编程语法即可快速上手编程。同时SDAA C
最大化地隐藏了底层硬件的实现细节，用户无需深入了解硬件的体系架构就可以正常使用太初AI加速卡提供的强大算
力，大幅度地降低学习成本，以便您可以专注于并行程序的开发。SDAA C的详细信息，可参考《SDAA C编程指
南》。

 SDAA异构并行计算平台

SDAA异构并行计算平台是太初元碁针对AI场景，自主研发的众核异构计算架构，能够充分发挥太初AI加速卡的计算能
力，为科学计算、人工智能等领域提供完整的解决方案。




                               图1 SDAA异构并行计算平台

SDAA异构并行计算平台以太初元碁人工智能系列硬件（太初AI加速卡）为基础，基础软件层完成对硬件驱动及操作系
统的高度抽象，并提供完整的运行时库，编程模型及配套的编译、调试和分析工具；加速库提供包括TecoDNN、
TecoBLAS等基础且丰富的算子库；应用层整合了当前主流的深度学习框架。

SDAA C运行于基础软件层，对下屏蔽了硬件的物理特性，对上提供了丰富的编程接口，以及代码调试、调优工具。用
户无需感知底层的硬件细节，仅需遵从SDAA C的编程规范即可使用太初AI加速卡，为高并发、大数据运算提供强有力

1.3.2 SDAA异构并行模型
SDAA异构并行模型采用主从异构的物理架构，即Host-Device运行模式。




                                 图1 SDAA异构并行模型

   主机端（Host端）：由CPU构成，主要负责对设备端（Device端）内存资源的申请和释放，并控制设备端

   设备端（Device端）：由太初AI加速卡构成，负责处理大规模的数据运算。太初AI加速卡通过高效的运算部件

   提高了数据处理的效率。

1.4 算子开发流程
算子开发的整体流程主要分为：算子分析、算子实现和编译运行三个阶段。




                                   图1 算子开发流程

  1. 算子分析：主要是分析算子的数学计算逻辑、输入输出数据，并明确算子的实现流程。

 2. 算子实现：实现算子计算逻辑的核心步骤，包括：

     a. 申请主机端内存，并搬运输入数据

     b. 申请Global存储空间，并搬运输入数据（主机端至Global）

     c. 申请SPM存储空间，并搬运输入数据（Global至SPM）到SPM进行计算，计算结果存储在SPM。

     d. 搬运输出数据（SPM至Global）

     e. 搬运输出数据（Global至主机端）

      f. 算子测试

 3. 编译运行：开发完成的算子，编译运行后可以获得目标结果。

领域、科学计算领域等，太初AI加速卡能够提升数据处理和模型训推的能力。

SDAA C相较于其他加速卡的编程语言，最大程度隐藏了底层硬件的实现细节，无需深入了解硬件的体系架构，就可以
正常使用太初AI加速卡提供的强大算力；支持C/C++编程语言，仅需学习少量的SDAA C编程语法，即可快速上手编
程。

 加法算子（C++）

传统的CPU计算一般以单线程为主，本文以计算512个浮点数据的加法为例，介绍如何在CPU实现add算子。

CPU完整代码如下：


   #include <stdio.h>
   #include <math.h>


   #define TOTAL_NUM (512)
   #define FSIZE (TOTAL_NUM * sizeof(float))


   int main()
   {
       // 主机端申请输入数据和输出数据空间
       float *hdata_x = (float *)malloc(FSIZE);
       float *hdata_y = (float *)malloc(FSIZE);
       float *hdata_z = (float *)malloc(FSIZE);
       // 初始化输入数据的值， 初始化方式自定
       for (int i = 0; i < TOTAL_NUM; i++) {
           hdata_x[i] = (float)i / TOTAL_NUM;
           hdata_y[i] = sin((float)i / TOTAL_NUM);
       }


       for (int i = 0; i < TOTAL_NUM; i++) {
           // 将初始化的数据循环逐个累加
           hdata_z[i] = hdata_x[i] + hdata_y[i];
       }


       // 数据利用完后，释放主机端内存空间
       free(hdata_x);

 加法算子（SDAA C）

SDAA C可以使用更多的线程，进行同时计算，每个线程计算一个数据，可以实现多线程、多数据的并行计算。

本文以32线程为例，代码如下：

   // 使用sdaa空间名
   using namespace sdaa;
   #define TOTAL_NUM (256 * 32)
   #define FSIZE (TOTAL_NUM * sizeof(float))


   // x，y：表示输入的Global存储空间地址，z：表示计算结果的Global存储空间地址
   __global__ void kernel_add(float *x, float *y, float *z, uint32_t total_num)
   {
       // 计算循环次数
       uint32_t tile_num = total_num / threadDim;
       // 所有计算核心SPE循环tile_num次，并行做加法计算
       for (uint32_t i = 0; i < tile_num; i += threadDim) {
           // threadIdx表示计算核心SPE的编号，用 i + threadIdx 做输入数据下x, y的角标，
           // 所有核心SPE并行计算不同的数据。
           z[i + threadIdx] = x[i + threadIdx] + y[i + threadIdx];
       }
   }


   int main()
   {
       // 设置0号SDAA计算设备，用于进行加法计算
       sdaaSetDevice(0);
       // 申请主机端输入数据和输出数据的内存空间
       float *hdata_x = (float *)malloc(FSIZE);
       float *hdata_y = (float *)malloc(FSIZE);
       float *hdata_z = (float *)malloc(FSIZE);
       // 初始化输入数据x，y的值， 初始化方式可以根据实际情况自行选择
       for (int i = 0; i < TOTAL_NUM; i++) {
           hdata_x[i] = (float)i / TOTAL_NUM;
           hdata_y[i] = sin((float)i / TOTAL_NUM);
       }
       // 定义设备端指针初始化为NULL
       float *dev_data_x = NULL;
       float *dev_data_y = NULL;
       float *dev_data_z = NULL;


       // 申请Global存储空间，并返回地址给之前定义的指针
       sdaaMalloc((void **)&dev_data_x, FSIZE);
       sdaaMalloc((void **)&dev_data_y, FSIZE);
       sdaaMalloc((void **)&dev_data_z, FSIZE);


       // 将主机端数据复制到Global存储空间；需要配置sdaaMemcpyHostToDevice，数据传输方向是主机端至设备端
       sdaaMemcpy(dev_data_x, hdata_x, FSIZE, sdaaMemcpyHostToDevice);

SDAA C遵循SPMD编程范式（详情请参考《SDAA C编程指南》的SPMD章节），同一计算核心阵列SPA内所有计算

     threadIdx ：当前计算核心SPE的ID号，可以通过该关键字为目标SPE分配计算任务。

   __global__ void kernel_add(float *x, float *y, float *z, uint32_t total_num)
   {

       for (uint32_t i = 0; i < tile_num; i++) {

           spm_z[i] = spm_x[i] + spm_y[i];

5.2.7 步骤六：算子测试
算子的精度测试，通常是在主机端（业内认可度高、精度高的硬件芯片，如：X86架构的高精度芯片）执行相同的计
算，将计算结果与设备端传回的计算结果进行比对验证，从而确定计算结果的精度。

本文主要介绍如何验证算子的精度。

 确定测试精度

测试精度通常根据算子的计算逻辑和输入输出数据类型，进行分析确认。

本示例的add加法算子，输入输出的数据类型是float（单精度浮点数），其单位精度是 2−23 ，约为1.19e-07。因
此，本文将绝对值误差定为5e-07。即：CPU计算的结果，和SDAA计算设备计算的结果之间的绝对值误差不超过 5e-
07。


       说明：

       您也可以根据实际情况对主机端和设备端的绝对值误差进行调整。


 精度调测

精度调测是将设备端输出的计算结果和主机端（X86架构的高精度芯片或业界其他标杆硬件）的计算数据结果进行逐一
对比。

   // 申请主机端的内存空间，用于存放主机端的加法计算结果
   float *golden_z = (float *)malloc(FSIZE);
   for (int i = 0; i < TOTAL_NUM; i++) {
       // 在主机端执行加法计算
       golden_z[i] = hdata_x[i] + hdata_y[i];
       // 主机端计算结果和设备端计算结果的绝对值误差如果大于5e-7，则提示精度不达标
       if(abs(golden_z[i] - hdata_z[i]) > 5e-7) {
           printf(" %dth value: %f is wrong! \n", i, hdata_z[i]);
           break;
       }
       if(i == TOTAL_NUM - 1) {
           printf(" SUCCESS! \n");
       }
   }

           if(abs(golden_z[i] - hdata_z[i]) > 5e-7) {

       free(hdata_x);
       free(hdata_y);

       sdaaFree(dev_data_y);

 确定算子名称（name）

矩阵乘算子的名称为 kernel_matmul ，即Kernel函数的名称为         kernel_matmul() 。


 确定算子任务

 确定算子数据结构(Tensor)

6.2 算子实现
6.2.1 设备端矩阵乘
 核心编程思路

  1. 申请SPM存储空间：使用SPM存储空间进行矩阵乘运算，性能更好。申请SPM存储空间，用于存储输入矩阵A和
   B，以及输出矩阵C的数据。

 2. 矩阵切分：将切分后的8次矩阵乘划分为4组，每次做2个矩阵乘，共循环4次，循环次数用j表示，取值范围0~3。

SDAA C零基础入门                                                                     开发矩阵乘算子




 3. 配置矩阵乘的参数，例如：是否需要刷出累加缓冲区数据，输入矩阵的行数偏移等，可参考SDAA C编程指南的非
   阻塞型矩阵乘（详情请参考《SDAA C编程指南》的配置型操作章节）章节。

 4. 加载权重矩阵B：矩阵B每次循环的起始元素为 Bk0 ，其中：第一个矩阵 k = (2 × j) × 32 ，第二个矩阵
                                             ​




    k = (2 × j + 1) × 32 。

   由矩阵B任意元素在内存中真实地址的计算公式 pAddrb = pBaseb + （k ⋅ N + n） 可以得出，两个矩阵起
                                                         ​       ​




   始元素在内存中的真实地址是：

    pBaseb + ((2 × j) × 32 × 32 + 0) 和 pBaseb + ((2 × j + 1) × 32 × 32 + 0) 。
              ​                                      ​




   获取到矩阵B起始元素在内存中的真实地址后，传入matmul_load_weight（详情请参考《SDAA C编程指南》
   的matmul_load_weight章节）接口，从SPM的指定地址加载权重矩阵到矩阵乘计算单元上。

 5. 加载输入矩阵A：矩阵A每次循环的起始元素为 A0k ，其中：第一个矩阵 k = (2 × j) × 32 ，第二个矩阵
                                                 ​




    k = (2 × j + 1) × 32 。

   由矩阵A任意元素在内存中真实地址的计算公式 pAddra = pBasea + （m ⋅ K + k ） 可以得出，两个矩阵
                                                             ​       ​




   起始元素在内存中的真实地址是：

    pBasea + (0 + (2 × j) × 32) 和 pBaseb + (0 + (2 × j + 1) × 32) 。
                  ​                      ​




   获取到矩阵A起始元素在内存中的真实地址后，传入matmul_compute（详情请参考《SDAA C编程指南》的
   matmul_compute章节）接口，从SPM的指定地址加载输入矩阵，并与加载好的权重矩阵做矩阵乘运算。


          说明：

          对于矩阵A1，第一行数据和第二行数据在内存里并不是连续的，所以在加载第一行以后的数据时，每次换
          行均需要跨过32 × 7个int16数据。而跨步设置的基本单位是64B，正好是32个int16数据，因此跨步的步
          长是 32 x 7 × 2B / 64B = 7。

 6. 计算结果累加并输出：将计算结果从累加缓冲区输出到SPM存储空间（matmul_store（详情请参考《SDAA C
   编程指南》的matmul_store章节）），再从SPM存储空间搬运到Global存储空间。

 代码实现

设备端矩阵乘的代码如下：

   #include <sdaa_matmul.h>


   using namespace sdaa;


   // B 数据batch大小
   #define B 32
   // M维度大小
   #define M 128
   // N维度大小
   #define N 32
   // K维度大小
   #define K 256
   // K维度切分的最小颗粒度
   #define K_BLK 32




文档版本：v1.1.0                     版权所有 © 太初(无锡)电子科技有限公司                                 23
SDAA C零基础入门                                                                                        开发矩阵乘算子




   // 矩阵乘的Kernel函数，参数是输入矩阵a，b和输出矩阵c的地址
   __global__ void kernel_matmul(char *dataA, char *dataB, char *dataC) {


      // 计算每个SPE 要平均分配多少batch的数据量
      int32_t batch_spe = B/threadDim;
      // 一个SPE分别被分得的 矩阵a, b, c的batch数的总数据量大小
      int32_t a_size_spe = batch_spe * M * K * sizeof(int16_t);
      int32_t b_size_spe = batch_spe * K * N * sizeof(int16_t);
      int32_t c_size_spe = batch_spe * M * N * sizeof(int32_t);
      // 给输入矩阵a（输入矩阵）申请SPM存储空间
      int16_t *spm_input = (int16_t *)malloc(a_size_spe);
      // 通过threadIdx做地址偏移，将当前spe分得的输入矩阵a的数据从Global内存上复制到
      // SPM上申请的a矩阵数据空间
      memcpy_async(spm_input, dataA + threadIdx * a_size_spe, a_size_spe, MemcpyGlobalToSpm);


      // 给输入矩阵b（权重矩阵）申请SPM存储空间
      int16_t *spm_weight = (int16_t *)malloc(b_size_spe);
      // 通过threadIdx做地址偏移，将当前spe分得的输入矩阵b的数据从Global内存上复制到
      // SPM上申请的b矩阵数据空间
      memcpy_async(spm_weight, dataB + threadIdx * b_size_spe, b_size_spe, MemcpyGlobalToSpm);
      // 等待矩阵a，b数据复制完成
      memcpy_wait();
      // 给输出矩阵c 申请SPM存储空间
      int32_t *spm_output = (int32_t *)malloc(c_size_spe);


      // 定义一个矩阵乘的句柄，可以将矩阵乘的一些配置项设置到句柄里。这样矩阵乘执行时，
      // 可以按照句柄里的配置来进行计算
      MatmulHandle handle;
      // 配置输入数据是short 类型，输出数据是int型
      matmul_init(handle, MatmulShortToInt);
      // 计算一次矩阵乘 K 的切分是32一块，一个循环计算2次。计算出循环次数
      uint32_t chunks = K / (K_BLK * 2);


      for (int j = 0; j < chunks; j++) {
           // 等待输入矩阵a矩阵数据加载完成，若之前没有加载动作，等待也会自动跳过
           matmul_wait_loading_input(handle);
           // 等待权重矩阵b矩阵数据加载完成，若之前没有加载动作，等待也会自动跳过
           matmul_wait_loading_weight(handle);
           // 设置本次matmul计算完成后，是否需要把数据从累加buffer 刷出到SPM上
           // 本次计算不是最后一次，所以不需要设置刷出数据
           matmul_set_flushing_output(handle, false);
           // 设置输入矩阵a的偏移量，通常是输入矩阵行数不够128，且没有从128行的第一行
           // 开始加载时，才需要设置的偏移量。此处输入是128行，且从128行里的第一行开始
           // 所以设置为0
           matmul_set_output_row_offset(handle, MatmulEnableOutputRowOffset, 0);
           // 加载权重矩阵b，由之前计算出循环的第一次地址是 pBase_b                + ((2 * j ) * 32 × 32 + 0)，
           // 对应到代码中是，spm_weight + K_BLK * K_BLK * j * 2
           matmul_load_weight(handle, spm_weight + K_BLK * K_BLK * j * 2, MatmulK32, MatmulN32);
           // 加载输入矩阵并进行计算，由之前计算出第一次地址是 pBase_a                 + (2 * j) * 32，
           // 对应到代码中是，spm_input + K_BLK * j * 2，注意跨步的基本单位是 64B，
           // 跨步的步长是7(即 K / K_BLK - 1)
           matmul_compute(handle, spm_input + K_BLK * j * 2, M, MatmulK32, (K / K_BLK - 1));


           // 设置本次matmul计算完成后，是否需要把数据从累加buffer 刷出到SPM上。
           // 当j == (chunks - 1)， 即最后一次计算后，需要把数据刷出到SPM上
           matmul_set_flushing_output(handle, (j == (chunks - 1)));
           // 设置输入矩阵a的偏移量，通常是输入矩阵行数不够128，且没有从128行的第一行
           // 开始加载时，才需要设置的偏移量。此处输入是128行，且从128行里的第一行开始




文档版本：v1.1.0                           版权所有 © 太初(无锡)电子科技有限公司                                              24
SDAA C零基础入门                                                                                   开发矩阵乘算子




           // 所以设置为0
           matmul_set_output_row_offset(handle, MatmulEnableOutputRowOffset, 0);
           // 加载权重矩阵b，由之前计算出循环的第一次地址是 pBase_b                + ((2 * j + 1) * 32 × 32 + 0)，
           // 对应到代码中是，spm_weight + K_BLK * K_BLK * (j * 2 + 1)
           matmul_load_weight(handle, spm_weight + K_BLK * K_BLK * (j * 2 + 1),
                              MatmulK32, MatmulN32);
           // 加载输入矩阵并进行计算，由之前计算出第一次地址是 pBase_a                 + (2 * j + 1) * 32，
           // 对应到代码中是，spm_input + K_BLK * (j * 2 + 1)，注意跨步的基本单位是 64B，
           // 跨步的步长是7(即 K / K_BLK - 1)
           matmul_compute(handle, spm_input + K_BLK * (j * 2 + 1), M, MatmulK32,
                           (K / K_BLK - 1));
       }
       // 把累加buffer 上的数据，输出到已申请的SPM地址 spm_output上
       matmul_store(handle, spm_output, M, MatmulN32);
       // 等待数据刷出完成
       matmul_wait(handle);
       // 通过threadIdx做地址偏移，将当前spe计算出的输出矩阵c的数据复制到Global上
       memcpy(dataC + threadIdx * c_size_spe, spm_output, c_size_spe);
       // 释放申请的SPM存储空间
       free(spm_output);
       free(spm_weight);
       free(spm_input);
   }

       int16_t *host_b = (int16_t *)malloc(b_size * sizeof(int16_t));

       // 在Host 端申请Host 自己计算的输出矩阵c 的对比数据的内存空间

       memset(true_c, 0, c_size * sizeof(int));

       // SPM上申请的a矩阵数据空间
       memcpy_async(spm_input, dataA + threadIdx * a_size_spe, a_size_spe, MemcpyGlobalToSpm);


       // 给输入矩阵b（权重矩阵）申请SPM存储空间
       int16_t *spm_weight = (int16_t *)malloc(b_size_spe);
       // 通过threadIdx做地址偏移，将当前spe分得的输入矩阵b的数据从Global内存上复制到
       // SPM上申请的b矩阵数据空间
       memcpy_async(spm_weight, dataB + threadIdx * b_size_spe, b_size_spe, MemcpyGlobalToSpm);
       // 等待矩阵a，b数据复制完成
       memcpy_wait();
       // 给输出矩阵c 申请SPM存储空间
       int32_t *spm_output = (int32_t *)malloc(c_size_spe);


       // 定义一个矩阵乘的句柄，可以将矩阵乘的一些配置项设置到句柄里。这样矩阵乘执行时，
       // 可以按照句柄里的配置来进行计算
       MatmulHandle handle;
       // 配置输入数据是short 类型，输出数据是int型
       matmul_init(handle, MatmulShortToInt);
       // 计算一次矩阵乘 K 的切分是32一块，一个循环计算2次。计算出循环次数
       uint32_t chunks = K / (K_BLK * 2);


       for (int j = 0; j < chunks; j++) {
           // 等待输入矩阵a矩阵数据加载完成，若之前没有加载动作，等待也会自动跳过
           matmul_wait_loading_input(handle);
           // 等待权重矩阵b矩阵数据加载完成，若之前没有加载动作，等待也会自动跳过
           matmul_wait_loading_weight(handle);
           // 设置本次matmul计算完成后，是否需要把数据从累加buffer 刷出到SPM上
           // 本次计算不是最后一次，所以不需要设置刷出数据
           matmul_set_flushing_output(handle, false);
           // 设置输入矩阵a的偏移量，通常是输入矩阵行数不够128，且没有从128行的第一行
           // 开始加载时，才需要设置的偏移量。此处输入是128行，且从128行里的第一行开始
           // 所以设置为0
           matmul_set_output_row_offset(handle, MatmulEnableOutputRowOffset, 0);
           // 加载权重矩阵b，由之前计算出循环的第一次地址是 pBase_b                 + ((2 * j ) * 32 × 32 + 0)，
           // 对应到代码中是，spm_weight + K_BLK * K_BLK * j * 2
           matmul_load_weight(handle, spm_weight + K_BLK * K_BLK * j * 2, MatmulK32, MatmulN32);
           // 加载输入矩阵并进行计算，由之前计算出第一次地址是 pBase_a                  + (2 * j) * 32，
           // 对应到代码中是，spm_input + K_BLK * j * 2，注意跨步的基本单位是 64B，
           // 跨步的步长是7(即 K / K_BLK - 1)
           matmul_compute(handle, spm_input + K_BLK * j * 2, M, MatmulK32, (K / K_BLK - 1));

           // 设置本次matmul计算完成后，是否需要把数据从累加buffer 刷出到SPM上。
           // 当j == (chunks - 1)， 即最后一次计算后，需要把数据刷出到SPM上
           matmul_set_flushing_output(handle, (j == (chunks - 1)));
           // 设置输入矩阵a的偏移量，通常是输入矩阵行数不够128，且没有从128行的第一行
           // 开始加载时，才需要设置的偏移量。此处输入是128行，且从128行里的第一行开始
           // 所以设置为0
           matmul_set_output_row_offset(handle, MatmulEnableOutputRowOffset, 0);
           // 加载权重矩阵b，由之前计算出循环的第一次地址是 pBase_b                   + ((2 * j + 1) * 32 × 32 + 0)，
           // 对应到代码中是，spm_weight + K_BLK * K_BLK * (j * 2 + 1)
           matmul_load_weight(handle, spm_weight + K_BLK * K_BLK * (j * 2 + 1),
                                MatmulK32, MatmulN32);
           // 加载输入矩阵并进行计算，由之前计算出第一次地址是 pBase_a                    + (2 * j + 1) * 32，
           // 对应到代码中是，spm_input + K_BLK * (j * 2 + 1)，注意跨步的基本单位是 64B，
           // 跨步的步长是7(即 K / K_BLK - 1)
           matmul_compute(handle, spm_input + K_BLK * (j * 2 + 1), M, MatmulK32,
                             (K / K_BLK - 1));
       }
       // 把累加buffer 上的数据，输出到已申请的SPM地址 spm_output上
       matmul_store(handle, spm_output, M, MatmulN32);
       // 等待数据刷出完成
       matmul_wait(handle);
       // 通过threadIdx做地址偏移，将当前spe计算出的输出矩阵c的数据复制到Global上
       memcpy(dataC + threadIdx * c_size_spe, spm_output, c_size_spe);
       // 释放申请的SPM存储空间
       free(spm_output);
       free(spm_weight);
       free(spm_input);
   }


   int main() {
       int b, i, j, k;


       char *dev_a = NULL;
       char *dev_b = NULL;
       char *dev_c = NULL;


       int a_size = B * M * K;
       int b_size = B * K * N;
       int c_size = B * M * N;


       unsigned int local_seed = time(NULL);


       // 设置需要用到的设备号
       sdaaSetDevice(1);
       // 在Device端申请输入矩阵a 的Global空间
       sdaaMalloc((void **)&dev_a, a_size * sizeof(int16_t));
       // 在Device端申请输入矩阵b 的Global空间
       sdaaMalloc((void **)&dev_b, b_size * sizeof(int16_t));
       // 在Device端申请输出矩阵c 的Global空间
       sdaaMalloc((void **)&dev_c, c_size * sizeof(int));
       // 在Host 端申请输入矩阵a的内存空间
       int16_t *host_a = (int16_t *)malloc(a_size * sizeof(int16_t));
       // 在Host 端申请输入矩阵b的内存空间
       int16_t *host_b = (int16_t *)malloc(b_size * sizeof(int16_t));
       // 在Host 端申请需要从设备端复制数据到Host端输出矩阵c的内存空间
       int *host_c = (int *)malloc(c_size * sizeof(int));
       // 在Host 端申请Host 自己计算的输出矩阵c 的对比数据的内存空间
       int *true_c = (int *)malloc(c_size * sizeof(int));
       // 初始化矩阵a, b的输入数据

      sdaaMemcpy(dev_b, host_b, b_size * sizeof(int16_t), sdaaMemcpyHostToDevice);

      kernel_matmul<<<1>>>(dev_a, dev_b, dev_c);

      sdaaMemcpy(host_c, dev_c, c_size * sizeof(int), sdaaMemcpyDeviceToHost);


      memset(true_c, 0, c_size * sizeof(int));
      // Host端计算 用于对比的输出矩阵c的数据
      for (b = 0; b < B; b++) {
          for (i = 0; i < M; i++) {

                  for (k = 0; k < K; k++) {
                        true_c[b * M * N + i * N + j] +=
                            ((int)host_a[b * M * K + i * K + k])
                             * ((int)host_b[b * K * N + k * N + j]);
                  }
              }

      for (i = 0; i < c_size; i++) {

              printf(
                  "error: find mismatching data on %dth value,     device: %d, host: %d\n",

      sdaaFree(dev_c);

7. 性能优化
算子的性能对模型训练、模型推理等均有较大影响。本文主要介绍如何基于SDAA C从计算和访存两个维度提升算子的
性能。

 计算相关

   增强指令或指令中数据的并行度，例如：使用向量指令，有效利用指令的流水等，具体可以参考《性能优化手册-
   算子篇》。
   增加计算数据的并行度，例如：使用多线程，多任务的并行计算，具体可以参考《性能优化手册-SDAA C篇》。

 访存相关

   从时间上提高访存的利用率，例如：非阻塞型的访存数据搬运，采用双流水的方式，减少数据的等待时间；增加
   内存搬运硬件的利用率，具体可以参考《性能优化手册-算子篇》。
   从数据空间上提高访存的利用率，提高数据的重复利用率，减少数据的搬运次数；通过数据广播的方式进行数据
   搬运，减少数据的重复搬运次数，具体可以参考《性能优化手册-SDAA C篇》。
