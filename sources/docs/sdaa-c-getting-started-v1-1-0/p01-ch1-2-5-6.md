# SDAA C 零基础入门 v1.1.0（第1部分）

## 1.3.1 SDAA 异构并行计算平台

SDAA 异构并行计算平台是太初元碁针对 AI 场景，自主研发的众核异构计算架构，能够充分发挥太初 AI 加速卡的计算能力，为科学计算、人工智能等领域提供完整的解决方案。

图1 SDAA 异构并行计算平台

SDAA 异构并行计算平台以太初元碁人工智能系列硬件（太初 AI 加速卡）为基础，基础软件层完成对硬件驱动及操作系统的高度抽象，并提供完整的运行时库，编程模型及配套的编译、调试和分析工具；加速库提供包括 TecoDNN、TecoBLAS 等基础且丰富的算子库；应用层整合了当前主流的深度学习框架。

SDAA C 运行于基础软件层，对下屏蔽了硬件的物理特性，对上提供了丰富的编程接口，以及代码调试、调优工具。用户无需感知底层的硬件细节，仅需遵从 SDAA C 的编程规范即可使用太初 AI 加速卡，为高并发、大数据运算提供强有力的算力支撑。

## 1.3.2 SDAA 异构并行模型

SDAA 异构并行模型采用主从异构的物理架构，即 Host-Device 运行模式。

图1 SDAA 异构并行模型

- **主机端（Host 端）**：由 CPU 构成，主要负责对设备端（Device 端）内存资源的申请和释放，并控制设备端（Device 端）处理任务的资源。
- **设备端（Device 端）**：由太初 AI 加速卡构成，负责处理大规模的数据运算。太初 AI 加速卡通过高效的运算部件（SPE）和访存部件（Global 存储器等）弥补了通用处理器在处理高并发、大数据场景下计算和访存上的短板，提高了数据处理的效率。
    - 太初 AI 加速卡由计算核心阵列 SPA（Synergistic Processor Element Array）和对应的 Global 高带宽存储器构成。
    - 每个计算核心阵列 SPA 是一个协从运算单元，由计算核心 SPE（Synergistic Processor Element）构成。

图2 太初 AI 加速卡

## 1.3.3 Kernel 函数

太初 AI 加速卡负责处理大规模的数据运算，计算核心 SPE 在 SDAA C 中相当于一个线程，用户可以使用 SPE 进行异构并行计算。

Kernel 函数是由 `__global__`（详情请参考《SDAA C 编程指南》的 `__global__` 章节）关键字修饰的函数，在设备端执行（类似于线程代码）。在使用 SDAA C 时，主机端通过调用 Kernel 函数将大量的计算任务分配给 SPE 执行，从而提高数据的处理效率。

图1 Kernel 函数

## 1.4 算子开发流程

算子开发的整体流程主要分为：算子分析、算子实现和编译运行三个阶段。

图1 算子开发流程

1. **算子分析**：主要是分析算子的数学计算逻辑、输入输出数据，并明确算子的实现流程。

2. **算子实现**：实现算子计算逻辑的核心步骤，包括：

    a. 申请主机端内存，并搬运输入数据

    b. 申请 Global 存储空间，并搬运输入数据（主机端至 Global）

    c. 申请 SPM 存储空间，并搬运输入数据（Global 至 SPM）到 SPM 进行计算，计算结果存储在 SPM。

    d. 搬运输出数据（SPM 至 Global）

    e. 搬运输出数据（Global 至主机端）

    f. 算子测试

3. **编译运行**：开发完成的算子，编译运行后可以获得目标结果。

## 4. 快速入门

相较于传统的 CPU 芯片，太初 AI 加速卡具有更多的计算单元，能够同时并行处理大量数据，提升数据计算的速度；同时太初 AI 加速卡具有更高的内存带宽。太初 AI 加速卡能够进行高速的矩阵乘法运算，对于依赖矩阵运算能力的深度学习领域、科学计算领域等，太初 AI 加速卡能够提升数据处理和模型训推的能力。

SDAA C 相较于其他加速卡的编程语言，最大程度隐藏了底层硬件的实现细节，无需深入了解硬件的体系架构，就可以正常使用太初 AI 加速卡提供的强大算力；支持 C/C++ 编程语言，仅需学习少量的 SDAA C 编程语法，即可快速上手编程。

图1 太初 AI 加速卡具有更多的计算单元

### 加法算子（C++）

传统的 CPU 计算一般以单线程为主，本文以计算 512 个浮点数据的加法为例，介绍如何在 CPU 实现 add 算子。

CPU 完整代码如下：

```cpp
#include <stdio.h>
#include <math.h>


#define TOTAL_NUM (512)
#define FSIZE (TOTAL_NUM * sizeof(float))


int main()
{
    // 主机端申请输入数据和输出数据空间
    float *hdata_x = (float *)malloc(FSIZE);
    float *hdata_y = (float *)malloc(FSIZE);
    float *hdata_z = (float *)malloc(FSIZE);
    // 初始化输入数据的值， 初始化方式自定
    for (int i = 0; i < TOTAL_NUM; i++) {
        hdata_x[i] = (float)i / TOTAL_NUM;
        hdata_y[i] = sin((float)i / TOTAL_NUM);
    }


    for (int i = 0; i < TOTAL_NUM; i++) {
        // 将初始化的数据循环逐个累加
        hdata_z[i] = hdata_x[i] + hdata_y[i];
    }


    // 数据利用完后，释放主机端内存空间
    free(hdata_x);
    free(hdata_y);
    free(hdata_z);


    printf(" DONE \n");
    return 0;
}
```

### 加法算子（SDAA C）

SDAA C 可以使用更多的线程，进行同时计算，每个线程计算一个数据，可以实现多线程、多数据的并行计算。

本文以 32 线程为例，代码如下：

```cpp
// 使用sdaa空间名
using namespace sdaa;
#define TOTAL_NUM (256 * 32)
#define FSIZE (TOTAL_NUM * sizeof(float))


// x，y：表示输入的Global存储空间地址，z：表示计算结果的Global存储空间地址
__global__ void kernel_add(float *x, float *y, float *z, uint32_t total_num)
{
    // 计算循环次数
    uint32_t tile_num = total_num / threadDim;
    // 所有计算核心SPE循环tile_num次，并行做加法计算
    for (uint32_t i = 0; i < tile_num; i += threadDim) {
        // threadIdx表示计算核心SPE的编号，用 i + threadIdx 做输入数据下x, y的角标，
        // 所有核心SPE并行计算不同的数据。
        z[i + threadIdx] = x[i + threadIdx] + y[i + threadIdx];
    }
}


int main()
{
    // 设置0号SDAA计算设备，用于进行加法计算
    sdaaSetDevice(0);
    // 申请主机端输入数据和输出数据的内存空间
    float *hdata_x = (float *)malloc(FSIZE);
    float *hdata_y = (float *)malloc(FSIZE);
    float *hdata_z = (float *)malloc(FSIZE);
    // 初始化输入数据x，y的值， 初始化方式可以根据实际情况自行选择
    for (int i = 0; i < TOTAL_NUM; i++) {
        hdata_x[i] = (float)i / TOTAL_NUM;
        hdata_y[i] = sin((float)i / TOTAL_NUM);
    }
    // 定义设备端指针初始化为NULL
    float *dev_data_x = NULL;
    float *dev_data_y = NULL;
    float *dev_data_z = NULL;


    // 申请Global存储空间，并返回地址给之前定义的指针
    sdaaMalloc((void **)&dev_data_x, FSIZE);
    sdaaMalloc((void **)&dev_data_y, FSIZE);
    sdaaMalloc((void **)&dev_data_z, FSIZE);


    // 将主机端数据复制到Global存储空间；需要配置sdaaMemcpyHostToDevice，数据传输方向是主机端至设备端
    sdaaMemcpy(dev_data_x, hdata_x, FSIZE, sdaaMemcpyHostToDevice);
    sdaaMemcpy(dev_data_y, hdata_y, FSIZE, sdaaMemcpyHostToDevice);


    // 调用Kernel函数，将实际输入数据、输出数据的地址填入函数参数，进行计算
    kernel_add<<<1>>>(dev_data_x, dev_data_y, dev_data_z, TOTAL_NUM);
    // 进行同步，等待所有SPE运行完Kernel函数
    sdaaDeviceSynchronize();
    // 将计算完成的数据从Global搬运到主机端内存
    sdaaMemcpy(hdata_z, dev_data_z, FSIZE, sdaaMemcpyDeviceToHost);


    // 数据使用完后，释放主机端内存
    free(hdata_x);
    free(hdata_y);
    free(hdata_z);
    // 数据使用完后，释放Global存储空间
    sdaaFree(dev_data_x);
    sdaaFree(dev_data_y);
    sdaaFree(dev_data_z);


    return 0;
}
```

## 5. 开发加法算子

### 5.1 算子分析

算子分析主要是分析算子的数学计算逻辑、输入输出数据，并明确算子的实现流程。本文以二维张量的加法为例，介绍如何进行算子分析。

#### 确定算子名称（name）

算子的名称需要体现算子的功能，本文的加法算子的名称以 `kernel_add` 为例，即 Kernel 函数的名称为 `kernel_add()`。

#### 明确算子任务

计算两个二维张量的加法，即将两个输入矩阵的每个数据进行一一对应的相加：

```
z = x + y
```

图1 加法计算逻辑

#### 确定算子数据结构（Tensor）

加法算子中有两组输入数据，一组输出数据。两组输入数据的名称分别为 `x`、`y`，输出数据的名称为 `z`；输入输出数据的格式为 2 维 (H,W) 张量，输入输出数据的形状信息为 (256,32)，即 256 行 32 列的矩阵。数据类型为单精度浮点型 float，详细信息如下：

| 输入输出 | 名称（name） | 形状（shape） | 数据类型（data type） | 数据排布格式（format） |
|----------|--------------|---------------|------------------------|------------------------|
| 输入数据 | x | (256,32) | float | (H,W) |
| 输入数据 | y | (256,32) | float | (H,W) |
| 输出数据 | z | (256,32) | float | (H,W) |

### 5.2 算子实现

#### 5.2.1 实现流程

算子实现是实现算子计算逻辑的核心步骤，其主要流程如下：

图1 算子开发流程

1. **申请主机端内存，并准备输入数据**：在主机端申请内存空间，并准备输入数据。

2. **申请 Global 存储空间，并传输数据（主机端至 Global）**：申请 Global 存储空间，并将输入数据从主机端搬运至 Global 存储空间。

3. **申请 SPM 存储空间，并传输数据（Global 至 SPM）到 SPM 进行计算**：申请 SPM 存储空间，将数据从 Global 存储空间搬运至 SPM 存储空间，进行加速计算，并得出计算结果。

4. **传输数据（SPM 至 Global）**：将计算结果从 SPM 存储空间搬运至 Global 存储空间。

5. **传输数据（Global 至主机端）**：将计算结果从 Global 存储空间搬运至主机端。

6. **算子测试**：算子的精度测试，通常是在主机端（业内认可度高、精度高的硬件芯片，如：X86 架构的高精度芯片）执行相同的计算，将计算结果与设备端传输回的计算结果进行比对验证。

> **说明：**
>
> 其中，第 1、2、5、6 步骤是主机端需要设计和执行的，代码都在主机端运行。而第 3、4 步骤是设备端需要设计和执行的，代码在设备端运行。

#### 5.2.2 步骤一：申请主机端内存并初始化输入数据

本文主要介绍如何申请主机端内存，以及如何初始化输入数据 `x`、`y`。

**编程思路**

1. 主机端代码结构与 CPU 端代码结构相同，从 `main()` 函数开始。

    ```cpp
    int main { }
    ```

2. 在 `main()` 函数中，设置设备端进行加法计算的 SDAA 计算设备编号，本文以 0 号 SDAA 计算设备为例。

    ```cpp
    int main {
        // 设置0号SDAA计算设备，用于进行加法计算
        sdaaSetDevice(0);
    }
    ```

3. 申请主机端输入数据和输出数据的内存空间。

    ```cpp
    float *hdata_x = (float *)malloc(FSIZE);
    float *hdata_y = (float *)malloc(FSIZE);
    float *hdata_z = (float *)malloc(FSIZE);
    ```

4. 初始化输入数据 `x`、`y`，初始化方式可以根据实际情况自行选择。

    ```cpp
    for (int i = 0; i < TOTAL_NUM; i++) {
        hdata_x[i] = (float)i / TOTAL_NUM;
        hdata_y[i] = sin((float)i / TOTAL_NUM);
    }
    ```

#### 5.2.3 步骤二：申请 Global 存储空间并搬运数据（主机端至 Global）

本文主要介绍如何申请 Global 存储空间，以及如何将输入数据 `x`、`y` 从主机端搬运至 Global 存储空间。

**编程思路**

1. 申请 Global 存储空间。

    ```cpp
    // 定义设备端指针初始化为NULL
    float *dev_data_x = NULL;
    float *dev_data_y = NULL;
    float *dev_data_z = NULL;


    // 申请Global存储空间，并返回地址给之前定义的指针
    sdaaMalloc((void **)&dev_data_x, FSIZE);
    sdaaMalloc((void **)&dev_data_y, FSIZE);
    sdaaMalloc((void **)&dev_data_z, FSIZE);
    ```

2. 将输入数据 `x`、`y` 从主机端搬运至 Global 存储空间。

    ```cpp
    // 将主机端数据复制到Global存储空间；需要配置sdaaMemcpyHostToDevice，数据传输方向是主机端至设备端
    sdaaMemcpy(dev_data_x, hdata_x, FSIZE, sdaaMemcpyHostToDevice);
    sdaaMemcpy(dev_data_y, hdata_y, FSIZE, sdaaMemcpyHostToDevice);
    ```

#### 5.2.4 步骤三：申请 SPM 存储空间并搬运数据（Global 至 SPM）至 SPM 进行计算

本文主要介绍如何申请 SPM 存储空间，并将数据从 Global 存储空间搬运至 SPM 存储空间进行加速计算，得出计算结果。

**数据切分**

SDAA C 遵循 SPMD 编程范式（详情请参考《SDAA C 编程指南》的 SPMD 章节），同一计算核心阵列 SPA 内所有计算核心 SPE 运行同一份应用程序，每个 SPE 可以独立完成对子问题的并行求解。其核心在于对复杂计算任务的切分，并合理地将计算任务分配到不同的 SPE 内进行计算。

图1 基于 SDAA C 的 SPMD 编程范式

SDAA C 提供了 threadIdx（详情请参考《SDAA C 编程指南》的 threadIdx 章节）和 threadDim（详情请参考《SDAA C 编程指南》的 threadDim 章节）为每个 SPE 定制计算任务。其中：

- `threadIdx`：当前计算核心 SPE 的 ID 号，可以通过该关键字为目标 SPE 分配计算任务。
- `threadDim`：同一计算核心阵列 SPA 内计算核心 SPE 的总数。

**编程思路**

1. 根据 SPE 的总数 `threadDim` 切分数据，每个 SPE 分得 `tile_num` 个 float 数据，并计算出每块数据的大小，单位：字节。

    ```cpp
    uint32_t tile_num = total_num / threadDim;
    uint32_t spm_size = tile_num * sizeof(float);
    ```

2. 申请 SPM 存储空间，大小为步骤一中每块数据的字节数目。

    ```cpp
    float *spm_x = (float *)malloc(spm_size);
    float *spm_y = (float *)malloc(spm_size);
    float *spm_z = (float *)malloc(spm_size);
    ```

3. 将数据从 Global 存储空间搬运至 SPM 存储空间。

    ```cpp
    memcpy(spm_x, (x + threadIdx * tile_num), spm_size);
    memcpy(spm_y, (y + threadIdx * tile_num), spm_size);
    ```

4. 每个 SPE 处理 `tile_num` 个 float 数据，且循环执行 `tile_num` 次的加法计算。

    ```cpp
    for (uint32_t i = 0; i < tile_num; i++) {
        spm_z[i] = spm_x[i] + spm_y[i];
    }
    ```

#### 5.2.5 步骤四：搬运数据（SPM 至 Global）

本文主要介绍如何将计算结果从 SPM 存储空间搬运至 Global 存储空间。

**编程思路**

将数据从 SPM 存储空间搬运至 Global 存储空间。

```cpp
memcpy((z + threadIdx * tile_num), spm_z, spm_size);
```

**设备端 Kernel 函数**

```cpp
// x，y：表示输入的Global地址，z：表示计算结果的Global地址
__global__ void kernel_add(float *x, float *y, float *z, uint32_t total_num)
{
    // 将所有数据切分为threadDim块，每块含有tile_num个数据，分给不同的SPE去计算
    uint32_t tile_num = total_num / threadDim;
    // 计算出tile_num个数据占用多少字节的数据空间
    uint32_t spm_size = tile_num * sizeof(float);
    // 在SPE为x, y, z申请SPM存储空间
    // 使用SPM存储空间可以加快数据的访问速度，提升性能
    float *spm_x = (float *)malloc(spm_size);
    float *spm_y = (float *)malloc(spm_size);
    float *spm_z = (float *)malloc(spm_size);
    // 使用输入数据x, y的首地址 + threadIdx * tile_num 偏移坐标作为搬运数据的源地址
    // 对Global存储空间的数据进行切分后，搬运到SPM存储空间
    memcpy(spm_x, (x + threadIdx * tile_num), spm_size);
    memcpy(spm_y, (y + threadIdx * tile_num), spm_size);


    // 每个计算核心SPE相隔tile_num个float数据，且循环tile_num次，进行加法计算
    for (uint32_t i = 0; i < tile_num; i++) {
        // 将SPM上所有输入数据相加，并将计算结果输出到对应的SPM存储空间
        spm_z[i] = spm_x[i] + spm_y[i];
    }
    // 用输出数据z的首地址 + threadIdx * tile_num 偏移坐标作为搬运数据的目的地址，
    // 将SPM存储空间上的计算结果搬运到Global存储空间
    memcpy((z + threadIdx * tile_num), spm_z, spm_size);
}
```

#### 5.2.6 步骤五：搬运数据（Global 至主机端）

本文主要介绍如何将计算结果从 Global 存储空间搬运至主机端。

**编程思路**

在主机端执行以下代码，将计算结果从 Global 存储空间搬运至主机端。

```cpp
sdaaMemcpy(hdata_z, dev_data_z, FSIZE, sdaaMemcpyDeviceToHost);
```

#### 5.2.7 步骤六：算子测试

算子的精度测试，通常是在主机端（业内认可度高、精度高的硬件芯片，如：X86 架构的高精度芯片）执行相同的计算，将计算结果与设备端传回的计算结果进行比对验证，从而确定计算结果的精度。

本文主要介绍如何验证算子的精度。

**确定测试精度**

测试精度通常根据算子的计算逻辑和输入输出数据类型，进行分析确认。

本示例的 add 加法算子，输入输出的数据类型是 float（单精度浮点数），其单位精度是 2⁻²³，约为 1.19e-07。因此，本文将绝对值误差定为 5e-07。即：CPU 计算的结果，和 SDAA 计算设备计算的结果之间的绝对值误差不超过 5e-07。

> **说明：**
>
> 您也可以根据实际情况对主机端和设备端的绝对值误差进行调整。

**精度调测**

精度调测是将设备端输出的计算结果和主机端（X86 架构的高精度芯片或业界其他标杆硬件）的计算数据结果进行逐一对比。

```cpp
// 申请主机端的内存空间，用于存放主机端的加法计算结果
float *golden_z = (float *)malloc(FSIZE);
for (int i = 0; i < TOTAL_NUM; i++) {
    // 在主机端执行加法计算
    golden_z[i] = hdata_x[i] + hdata_y[i];
    // 主机端计算结果和设备端计算结果的绝对值误差如果大于5e-7，则提示精度不达标
    if(abs(golden_z[i] - hdata_z[i]) > 5e-7) {
        printf(" %dth value: %f is wrong! \n", i, hdata_z[i]);
        break;
    }
    if(i == TOTAL_NUM - 1) {
        printf(" SUCCESS! \n");
    }
}
```

> **说明：**
>
> - 可以通过数据的标号确认精度存在误差的数据。
> - 如果绝对值误差不达标（大于 5e-7），可以从计算和数据存储两个主要维度，分析问题的原因：
>     - **计算：**
>         - 查看算子的计算逻辑是否有误。
>         - 使用的计算指令是否正确恰当。如果指令本身有问题或者使用有问题，也会引起精度问题。
>     - **数据存储：**
>         - 检查使用的数据地址是否正确。
>         - 异步数据操作时，操作同步不当，引起的数据踩踏的问题，也会导致精度错误。

#### 5.2.8 完整程序

加法算子 `kernel_add()` 的完整程序如下：

```cpp
// 使用sdaa空间名
using namespace sdaa;
#define TOTAL_NUM (256 * 32)
#define FSIZE (TOTAL_NUM * sizeof(float))


// x，y：表示输入的Global地址，z：表示计算结果的Global地址
__global__ void kernel_add(float *x, float *y, float *z, uint32_t total_num)
{
    // 将所有数据切分为threadDim块，每块含有tile_num个数据，分给不同的SPE去计算
    uint32_t tile_num = total_num / threadDim;
    // 计算出tile_num个数据占用多少字节的数据空间
    uint32_t spm_size = tile_num * sizeof(float);
    // 在SPE为x，y，z申请SPM存储空间
    // 使用SPM存储空间可以加快数据的访问速度，提升性能
    float *spm_x = (float *)malloc(spm_size);
    float *spm_y = (float *)malloc(spm_size);
    float *spm_z = (float *)malloc(spm_size);
    // 使用输入数据x, y的首地址 + threadIdx * tile_num 偏移坐标作为搬运数据的源地址
    // 对Global存储空间的数据进行切分后，搬运到SPM存储空间
    memcpy(spm_x, (x + threadIdx * tile_num), spm_size);
    memcpy(spm_y, (y + threadIdx * tile_num), spm_size);


    // 每个计算核心SPE相隔tile_num个float数据，且循环tile_num次，进行加法计算
    for (uint32_t i = 0; i < tile_num; i++) {
        // 将SPM上所有输入数据相加，并将计算结果输出到对应的SPM存储空间
        spm_z[i] = spm_x[i] + spm_y[i];
    }
    // 用输出数据z的首地址 + threadIdx * tile_num 偏移坐标作为搬运数据的目的地址
    // 将SPM存储空间上的计算结果搬运到Global存储空间
    memcpy((z + threadIdx * tile_num), spm_z, spm_size);
}


int main()
{
    // 设置0号SDAA计算设备，用于进行加法计算
    sdaaSetDevice(0);
    // 申请主机端输入数据和输出数据的内存空间
    float *hdata_x = (float *)malloc(FSIZE);
    float *hdata_y = (float *)malloc(FSIZE);
    float *hdata_z = (float *)malloc(FSIZE);
    // 初始化输入数据x，y的值， 初始化方式可以根据实际情况自行选择
    for (int i = 0; i < TOTAL_NUM; i++) {
        hdata_x[i] = (float)(i / TOTAL_NUM);
        hdata_y[i] = (float)sin(i / TOTAL_NUM);
    }
    // 定义设备端指针初始化为NULL
    float *dev_data_x = NULL;
    float *dev_data_y = NULL;
    float *dev_data_z = NULL;


    // 申请Global存储空间，并返回地址给之前定义的指针
    sdaaMalloc((void **)&dev_data_x, FSIZE);
    sdaaMalloc((void **)&dev_data_y, FSIZE);
    sdaaMalloc((void **)&dev_data_z, FSIZE);


    // 将主机端数据复制到Global存储空间；需要配置sdaaMemcpyHostToDevice，数据传输方向是主机端至设备端
    sdaaMemcpy(dev_data_x, hdata_x, FSIZE, sdaaMemcpyHostToDevice);
    sdaaMemcpy(dev_data_y, hdata_y, FSIZE, sdaaMemcpyHostToDevice);


    // 调用Kernel函数，将实际输入数据、输出数据的地址填入函数参数，进行计算
    kernel_add<<<1>>>(dev_data_x, dev_data_y, dev_data_z, TOTAL_NUM);
    // 进行同步，等待所有SPE运行完Kernel函数
    sdaaDeviceSynchronize();
    // 将计算完成的数据从Global搬运到主机端内存
    sdaaMemcpy(hdata_z, dev_data_z, FSIZE, sdaaMemcpyDeviceToHost);


    // 申请主机端的内存空间，用于存放主机端的加法计算结果
    float *golden_z = (float *)malloc(FSIZE);
    for (int i = 0; i < TOTAL_NUM; i++) {
        // 在主机端执行加法计算
        golden_z[i] = hdata_x[i] + hdata_y[i];
        // 主机端计算结果和设备端计算结果的绝对值误差如果大于5e-7，则提示精度不达标
        if(abs(golden_z[i] - hdata_z[i]) > 5e-7) {
            printf(" %dth value: %f is wrong! \n", i, hdata_z[i]);
            break;
        }
        if(i == TOTAL_NUM - 1) {
            printf(" SUCCESS! \n");
        }
    }


    // 数据使用完后，释放主机端内存
    free(hdata_x);
    free(hdata_y);
    free(hdata_z);
    free(golden_z);
    // 数据使用完后，释放Global存储空间
    sdaaFree(dev_data_x);
    sdaaFree(dev_data_y);
    sdaaFree(dev_data_z);


    return 0;
}
```

## 6. 开发矩阵乘算子

### 6.1 算子分析

本例将在太初 AI 加速卡上使用 SDAA C 开发矩阵乘算子。

#### 确定算子名称（name）

矩阵乘算子的名称为 `kernel_matmul`，即 Kernel 函数的名称为 `kernel_matmul()`。

#### 确定算子任务

计算两个矩阵的乘积，计算公式：

```
C = A × B
```

#### 确定算子数据结构（Tensor）

矩阵乘法算子中有两组输入数据，一组输出数据。输入矩阵分别为 A 和 B，输出矩阵为 C，详细信息如下：

| 输入输出 | 名称（name） | 形状（shape） | 数据类型（data type） | 数据排布格式（format） |
|----------|--------------|---------------|------------------------|------------------------|
| 输入矩阵 | A | (32,128,256) | int16 | (N, H, W) |
| 输入矩阵 | B | (32,256,32) | int16 | (N, H, W) |
| 输出矩阵 | C | (32,128,32) | int16 | (N, H, W) |

#### 算子实现逻辑分析

**矩阵切分**

为提升矩阵乘算子的计算性能，可以对矩阵 A 和 B 进行数据切分。对于单个 Batch（1,H,W），将矩阵 A 和矩阵 B 切分后：

图1 矩阵切分

其中：

- A 矩阵 (128,256)，分为 8 个 (128,32) 的小矩阵；B 矩阵 (256,32) 分为 8 个 (32,32) 的小矩阵。A 和 B 矩阵切分后：
    - A 矩阵，每次加载的数据量是 m × k，其中：行数 m 是 128，列数 k 是 32。
    - B 矩阵，每次加载的数据量是 k × n，其中：行数 k 是 32，列数 n 是 32。
- 切分后的矩阵相乘的计算结果会累加，并缓存在矩阵乘计算单元的累加缓冲区内。

**坐标转换**

矩阵是二维向量，而在实际内存中数据的排布是一维线性的。因此，在进行二维矩阵乘法时，需要将二维矩阵的元素坐标换算为内存中的真实地址。

例如：

- 矩阵 A 的维度是 (M, K)，某一矩阵元素 A_mk 的坐标是 (m,k)，A 矩阵第一行第一列的元素 A00 在内存中的地址为 pBase_a，则 A_mk 在内存中的真实地址：

    ```
    pAddr_a = pBase_a + （m · K + k）
    ```

- 矩阵 B 的维度是 (K,N)，某一矩阵元素 B_kn 的坐标是 (k,n)，B 矩阵第一行第一列的元素 B00 在内存中的地址为 pBase_b，则 B_kn 在内存中的真实地址：

    ```
    pAddr_b = pBase_b + （k · N + n）
    ```

### 6.2 算子实现

#### 6.2.1 设备端矩阵乘

**核心编程思路**

1. **申请 SPM 存储空间**：使用 SPM 存储空间进行矩阵乘运算，性能更好。申请 SPM 存储空间，用于存储输入矩阵 A 和 B，以及输出矩阵 C 的数据。

2. **矩阵切分**：将切分后的 8 次矩阵乘划分为 4 组，每次做 2 个矩阵乘，共循环 4 次，循环次数用 j 表示，取值范围 0~3。

3. **配置矩阵乘的参数**，例如：是否需要刷出累加缓冲区数据，输入矩阵的行数偏移等，可参考 SDAA C 编程指南的非阻塞型矩阵乘（详情请参考《SDAA C 编程指南》的配置型操作章节）章节。

4. **加载权重矩阵 B**：矩阵 B 每次循环的起始元素为 B_k0，其中：第一个矩阵 k = (2 × j) × 32，第二个矩阵 k = (2 × j + 1) × 32。

    由矩阵 B 任意元素在内存中真实地址的计算公式 `pAddr_b = pBase_b + （k · N + n）` 可以得出，两个矩阵起始元素在内存中的真实地址是：

    ```
    pBase_b + ((2 × j) × 32 × 32 + 0) 和 pBase_b + ((2 × j + 1) × 32 × 32 + 0)
    ```

    获取到矩阵 B 起始元素在内存中的真实地址后，传入 matmul_load_weight（详情请参考《SDAA C 编程指南》的 matmul_load_weight 章节）接口，从 SPM 的指定地址加载权重矩阵到矩阵乘计算单元上。

5. **加载输入矩阵 A**：矩阵 A 每次循环的起始元素为 A_0k，其中：第一个矩阵 k = (2 × j) × 32，第二个矩阵 k = (2 × j + 1) × 32。

    由矩阵 A 任意元素在内存中真实地址的计算公式 `pAddr_a = pBase_a + （m · K + k）` 可以得出，两个矩阵起始元素在内存中的真实地址是：

    ```
    pBase_a + (0 + (2 × j) × 32) 和 pBase_b + (0 + (2 × j + 1) × 32)
    ```

    获取到矩阵 A 起始元素在内存中的真实地址后，传入 matmul_compute（详情请参考《SDAA C 编程指南》的 matmul_compute 章节）接口，从 SPM 的指定地址加载输入矩阵，并与加载好的权重矩阵做矩阵乘运算。

    > **说明：**
    >
    > 对于矩阵 A1，第一行数据和第二行数据在内存里并不是连续的，所以在加载第一行以后的数据时，每次换行均需要跨过 32 × 7 个 int16 数据。而跨步设置的基本单位是 64B，正好是 32 个 int16 数据，因此跨步的步长是 32 x 7 × 2B / 64B = 7。

6. **计算结果累加并输出**：将计算结果从累加缓冲区输出到 SPM 存储空间（matmul_store（详情请参考《SDAA C 编程指南》的 matmul_store 章节）），再从 SPM 存储空间搬运到 Global 存储空间。

**代码实现**

设备端矩阵乘的代码如下：

```cpp
#include <sdaa_matmul.h>


using namespace sdaa;


// B 数据batch大小
#define B 32
// M维度大小
#define M 128
// N维度大小
#define N 32
// K维度大小
#define K 256
// K维度切分的最小颗粒度
#define K_BLK 32


// 矩阵乘的Kernel函数，参数是输入矩阵a，b和输出矩阵c的地址
__global__ void kernel_matmul(char *dataA, char *dataB, char *dataC) {


   // 计算每个SPE 要平均分配多少batch的数据量
   int32_t batch_spe = B/threadDim;
   // 一个SPE分别被分得的 矩阵a, b, c的batch数的总数据量大小
   int32_t a_size_spe = batch_spe * M * K * sizeof(int16_t);
   int32_t b_size_spe = batch_spe * K * N * sizeof(int16_t);
   int32_t c_size_spe = batch_spe * M * N * sizeof(int32_t);
   // 给输入矩阵a（输入矩阵）申请SPM存储空间
   int16_t *spm_input = (int16_t *)malloc(a_size_spe);
   // 通过threadIdx做地址偏移，将当前spe分得的输入矩阵a的数据从Global内存上复制到
   // SPM上申请的a矩阵数据空间
   memcpy_async(spm_input, dataA + threadIdx * a_size_spe, a_size_spe, MemcpyGlobalToSpm);


   // 给输入矩阵b（权重矩阵）申请SPM存储空间
   int16_t *spm_weight = (int16_t *)malloc(b_size_spe);
   // 通过threadIdx做地址偏移，将当前spe分得的输入矩阵b的数据从Global内存上复制到
   // SPM上申请的b矩阵数据空间
   memcpy_async(spm_weight, dataB + threadIdx * b_size_spe, b_size_spe, MemcpyGlobalToSpm);
   // 等待矩阵a，b数据复制完成
   memcpy_wait();
   // 给输出矩阵c 申请SPM存储空间
   int32_t *spm_output = (int32_t *)malloc(c_size_spe);


   // 定义一个矩阵乘的句柄，可以将矩阵乘的一些配置项设置到句柄里。这样矩阵乘执行时，
   // 可以按照句柄里的配置来进行计算
   MatmulHandle handle;
   // 配置输入数据是short 类型，输出数据是int型
   matmul_init(handle, MatmulShortToInt);
   // 计算一次矩阵乘 K 的切分是32一块，一个循环计算2次。计算出循环次数
   uint32_t chunks = K / (K_BLK * 2);


   for (int j = 0; j < chunks; j++) {
        // 等待输入矩阵a矩阵数据加载完成，若之前没有加载动作，等待也会自动跳过
        matmul_wait_loading_input(handle);
        // 等待权重矩阵b矩阵数据加载完成，若之前没有加载动作，等待也会自动跳过
        matmul_wait_loading_weight(handle);
        // 设置本次matmul计算完成后，是否需要把数据从累加buffer 刷出到SPM上
        // 本次计算不是最后一次，所以不需要设置刷出数据
        matmul_set_flushing_output(handle, false);
        // 设置输入矩阵a的偏移量，通常是输入矩阵行数不够128，且没有从128行的第一行
        // 开始加载时，才需要设置的偏移量。此处输入是128行，且从128行里的第一行开始
        // 所以设置为0
        matmul_set_output_row_offset(handle, MatmulEnableOutputRowOffset, 0);
        // 加载权重矩阵b，由之前计算出循环的第一次地址是 pBase_b + ((2 * j ) * 32 × 32 + 0)，
        // 对应到代码中是，spm_weight + K_BLK * K_BLK * j * 2
        matmul_load_weight(handle, spm_weight + K_BLK * K_BLK * j * 2, MatmulK32, MatmulN32);
        // 加载输入矩阵并进行计算，由之前计算出第一次地址是 pBase_a + (2 * j) * 32，
        // 对应到代码中是，spm_input + K_BLK * j * 2，注意跨步的基本单位是 64B，
        // 跨步的步长是7(即 K / K_BLK - 1)
        matmul_compute(handle, spm_input + K_BLK * j * 2, M, MatmulK32, (K / K_BLK - 1));


        // 设置本次matmul计算完成后，是否需要把数据从累加buffer 刷出到SPM上。
        // 当j == (chunks - 1)， 即最后一次计算后，需要把数据刷出到SPM上
        matmul_set_flushing_output(handle, (j == (chunks - 1)));
        // 设置输入矩阵a的偏移量，通常是输入矩阵行数不够128，且没有从128行的第一行
        // 开始加载时，才需要设置的偏移量。此处输入是128行，且从128行里的第一行开始
        // 所以设置为0
        matmul_set_output_row_offset(handle, MatmulEnableOutputRowOffset, 0);
        // 加载权重矩阵b，由之前计算出循环的第一次地址是 pBase_b + ((2 * j + 1) * 32 × 32 + 0)，
        // 对应到代码中是，spm_weight + K_BLK * K_BLK * (j * 2 + 1)
        matmul_load_weight(handle, spm_weight + K_BLK * K_BLK * (j * 2 + 1),
                           MatmulK32, MatmulN32);
        // 加载输入矩阵并进行计算，由之前计算出第一次地址是 pBase_a + (2 * j + 1) * 32，
        // 对应到代码中是，spm_input + K_BLK * (j * 2 + 1)，注意跨步的基本单位是 64B，
        // 跨步的步长是7(即 K / K_BLK - 1)
        matmul_compute(handle, spm_input + K_BLK * (j * 2 + 1), M, MatmulK32,
                        (K / K_BLK - 1));
    }
    // 把累加buffer 上的数据，输出到已申请的SPM地址 spm_output上
    matmul_store(handle, spm_output, M, MatmulN32);
    // 等待数据刷出完成
    matmul_wait(handle);
    // 通过threadIdx做地址偏移，将当前spe计算出的输出矩阵c的数据复制到Global上
    memcpy(dataC + threadIdx * c_size_spe, spm_output, c_size_spe);
    // 释放申请的SPM存储空间
    free(spm_output);
    free(spm_weight);
    free(spm_input);
}
```

#### 6.2.2 主机端矩阵乘

在主机端同样做一次矩阵 A 和矩阵 B 的乘法计算，生成用于对比的数据。主机端矩阵乘的代码如下：

```cpp
#include <time.h>


// B 数据batch大小
#define B 32
// M维度大小
#define M 128
// N维度大小
#define N 32
// K维度大小
#define K 256
// K维度切分的最小颗粒度
#define K_BLK 32


int main() {
    int b, i, j, k;


    int a_size = B * M * K;
    int b_size = B * K * N;
    int c_size = B * M * N;


    unsigned int local_seed = time(NULL);


    // 在Host 端申请输入矩阵a的内存空间
    int16_t *host_a = (int16_t *)malloc(a_size * sizeof(int16_t));
    // 在Host 端申请输入矩阵b的内存空间
    int16_t *host_b = (int16_t *)malloc(b_size * sizeof(int16_t));
    // 在Host 端申请需要从设备端复制数据到Host端输出矩阵c的内存空间
    int *host_c = (int *)malloc(c_size * sizeof(int));


    // 在Host 端申请Host 自己计算的输出矩阵c 的对比数据的内存空间
    int *true_c = (int *)malloc(c_size * sizeof(int));
    // 初始化矩阵a, b的输入数据
    memset(host_a, 0, a_size * sizeof(int16_t));
    memset(host_b, 0, b_size * sizeof(int16_t));
    for (b = 0; b < B; b++) {
        for (i = 0; i < M; i++) {
            for (j = 0; j < K; j++) {
                host_a[b * M * K + i * K + j] = rand_r(&local_seed) % 100 - 50;
            }
        }
        for (i = 0; i < K; i++) {
            for (j = 0; j < N; j++) {
                host_b[b * K * N + i * N + j] = rand_r(&local_seed) % 5 - 2;
            }
        }
    }


    memset(true_c, 0, c_size * sizeof(int));
    // Host端计算 用于对比的输出矩阵c的数据
    for (b = 0; b < B; b++) {
        for (i = 0; i < M; i++) {
            for (j = 0; j < N; j++) {
                for (k = 0; k < K; k++) {
                    true_c[b * M * N + i * N + j] +=
                        ((int)host_a[b * M * K + i * K + k])
                           * ((int)host_b[b * K * N + k * N + j]);
                }
            }
        }
    }


    // 释放Host端内存空间
    free(true_c);
    free(host_c);
    free(host_b);
    free(host_a);


    return 0;
}
```

#### 6.2.3 完整程序

将设备端计算出的结果复制到主机端，并和主机端生成数据，进行逐一对比。完整代码如下：

```cpp
#include <time.h>
#include <sdaa_matmul.h>


using namespace sdaa;


// B 数据batch大小
#define B 32
// M维度大小
#define M 128
// N维度大小
#define N 32
// K维度大小
#define K 256
// K维度切分的最小颗粒度
#define K_BLK 32


// 矩阵乘的核函数，参数是输入矩阵a，b和输出矩阵c的地址
__global__ void kernel_matmul(char *dataA, char *dataB, char *dataC) {


    // 计算每个SPE 要平均分配多少batch的数据量
    int32_t batch_spe = B/threadDim;
    // 一个SPE分别被分得的 矩阵a, b, c的batch数的总数据量大小
    int32_t a_size_spe = batch_spe * M * K * sizeof(int16_t);
    int32_t b_size_spe = batch_spe * K * N * sizeof(int16_t);
    int32_t c_size_spe = batch_spe * M * N * sizeof(int32_t);
    // 给输入矩阵a（输入矩阵）申请SPM存储空间
    int16_t *spm_input = (int16_t *)malloc(a_size_spe);
    // 通过threadIdx做地址偏移，将当前spe分得的输入矩阵a的数据从Global内存上复制到
    // SPM上申请的a矩阵数据空间
    memcpy_async(spm_input, dataA + threadIdx * a_size_spe, a_size_spe, MemcpyGlobalToSpm);


    // 给输入矩阵b（权重矩阵）申请SPM存储空间
    int16_t *spm_weight = (int16_t *)malloc(b_size_spe);
    // 通过threadIdx做地址偏移，将当前spe分得的输入矩阵b的数据从Global内存上复制到
    // SPM上申请的b矩阵数据空间
    memcpy_async(spm_weight, dataB + threadIdx * b_size_spe, b_size_spe, MemcpyGlobalToSpm);
    // 等待矩阵a，b数据复制完成
    memcpy_wait();
    // 给输出矩阵c 申请SPM存储空间
    int32_t *spm_output = (int32_t *)malloc(c_size_spe);


    // 定义一个矩阵乘的句柄，可以将矩阵乘的一些配置项设置到句柄里。这样矩阵乘执行时，
    // 可以按照句柄里的配置来进行计算
    MatmulHandle handle;
    // 配置输入数据是short 类型，输出数据是int型
    matmul_init(handle, MatmulShortToInt);
    // 计算一次矩阵乘 K 的切分是32一块，一个循环计算2次。计算出循环次数
    uint32_t chunks = K / (K_BLK * 2);


    for (int j = 0; j < chunks; j++) {
        // 等待输入矩阵a矩阵数据加载完成，若之前没有加载动作，等待也会自动跳过
        matmul_wait_loading_input(handle);
        // 等待权重矩阵b矩阵数据加载完成，若之前没有加载动作，等待也会自动跳过
        matmul_wait_loading_weight(handle);
        // 设置本次matmul计算完成后，是否需要把数据从累加buffer 刷出到SPM上
        // 本次计算不是最后一次，所以不需要设置刷出数据
        matmul_set_flushing_output(handle, false);
        // 设置输入矩阵a的偏移量，通常是输入矩阵行数不够128，且没有从128行的第一行
        // 开始加载时，才需要设置的偏移量。此处输入是128行，且从128行里的第一行开始
        // 所以设置为0
        matmul_set_output_row_offset(handle, MatmulEnableOutputRowOffset, 0);
        // 加载权重矩阵b，由之前计算出循环的第一次地址是 pBase_b + ((2 * j ) * 32 × 32 + 0)，
        // 对应到代码中是，spm_weight + K_BLK * K_BLK * j * 2
        matmul_load_weight(handle, spm_weight + K_BLK * K_BLK * j * 2, MatmulK32, MatmulN32);
        // 加载输入矩阵并进行计算，由之前计算出第一次地址是 pBase_a + (2 * j) * 32，
        // 对应到代码中是，spm_input + K_BLK * j * 2，注意跨步的基本单位是 64B，
        // 跨步的步长是7(即 K / K_BLK - 1)
        matmul_compute(handle, spm_input + K_BLK * j * 2, M, MatmulK32, (K / K_BLK - 1));


        // 设置本次matmul计算完成后，是否需要把数据从累加buffer 刷出到SPM上。
        // 当j == (chunks - 1)， 即最后一次计算后，需要把数据刷出到SPM上
        matmul_set_flushing_output(handle, (j == (chunks - 1)));
        // 设置输入矩阵a的偏移量，通常是输入矩阵行数不够128，且没有从128行的第一行
        // 开始加载时，才需要设置的偏移量。此处输入是128行，且从128行里的第一行开始
        // 所以设置为0
        matmul_set_output_row_offset(handle, MatmulEnableOutputRowOffset, 0);
        // 加载权重矩阵b，由之前计算出循环的第一次地址是 pBase_b + ((2 * j + 1) * 32 × 32 + 0)，
        // 对应到代码中是，spm_weight + K_BLK * K_BLK * (j * 2 + 1)
        matmul_load_weight(handle, spm_weight + K_BLK * K_BLK * (j * 2 + 1),
                             MatmulK32, MatmulN32);
        // 加载输入矩阵并进行计算，由之前计算出第一次地址是 pBase_a + (2 * j + 1) * 32，
        // 对应到代码中是，spm_input + K_BLK * (j * 2 + 1)，注意跨步的基本单位是 64B，
        // 跨步的步长是7(即 K / K_BLK - 1)
        matmul_compute(handle, spm_input + K_BLK * (j * 2 + 1), M, MatmulK32,
                          (K / K_BLK - 1));
    }
    // 把累加buffer 上的数据，输出到已申请的SPM地址 spm_output上
    matmul_store(handle, spm_output, M, MatmulN32);
    // 等待数据刷出完成
    matmul_wait(handle);
    // 通过threadIdx做地址偏移，将当前spe计算出的输出矩阵c的数据复制到Global上
    memcpy(dataC + threadIdx * c_size_spe, spm_output, c_size_spe);
    // 释放申请的SPM存储空间
    free(spm_output);
    free(spm_weight);
    free(spm_input);
}


int main() {
    int b, i, j, k;


    char *dev_a = NULL;
    char *dev_b = NULL;
    char *dev_c = NULL;


    int a_size = B * M * K;
    int b_size = B * K * N;
    int c_size = B * M * N;


    unsigned int local_seed = time(NULL);


    // 设置需要用到的设备号
    sdaaSetDevice(1);
    // 在Device端申请输入矩阵a 的Global空间
    sdaaMalloc((void **)&dev_a, a_size * sizeof(int16_t));
    // 在Device端申请输入矩阵b 的Global空间
    sdaaMalloc((void **)&dev_b, b_size * sizeof(int16_t));
    // 在Device端申请输出矩阵c 的Global空间
    sdaaMalloc((void **)&dev_c, c_size * sizeof(int));
    // 在Host 端申请输入矩阵a的内存空间
    int16_t *host_a = (int16_t *)malloc(a_size * sizeof(int16_t));
    // 在Host 端申请输入矩阵b的内存空间
    int16_t *host_b = (int16_t *)malloc(b_size * sizeof(int16_t));
    // 在Host 端申请需要从设备端复制数据到Host端输出矩阵c的内存空间
    int *host_c = (int *)malloc(c_size * sizeof(int));
    // 在Host 端申请Host 自己计算的输出矩阵c 的对比数据的内存空间
    int *true_c = (int *)malloc(c_size * sizeof(int));
    // 初始化矩阵a, b的输入数据
    memset(host_a, 0, a_size * sizeof(int16_t));
    memset(host_b, 0, b_size * sizeof(int16_t));
    for (b = 0; b < B; b++) {
        for (i = 0; i < M; i++) {
            for (j = 0; j < K; j++) {
                host_a[b * M * K + i * K + j] = rand_r(&local_seed) % 100 - 50;
            }
        }
        for (i = 0; i < K; i++) {
            for (j = 0; j < N; j++) {
                host_b[b * K * N + i * N + j] = rand_r(&local_seed) % 5 - 2;
            }
        }
    }


    // 将初始化好的输入矩阵a数据从Host 端复制到Device 端
    sdaaMemcpy(dev_a, host_a, a_size * sizeof(int16_t), sdaaMemcpyHostToDevice);
    // 将初始化好的输入矩阵b数据从Host 端复制到Device 端
    sdaaMemcpy(dev_b, host_b, b_size * sizeof(int16_t), sdaaMemcpyHostToDevice);


    // Kernel函数kernel_matmul运行
    kernel_matmul<<<1>>>(dev_a, dev_b, dev_c);
    sdaaDeviceSynchronize();
    // 将Device 端计算好的 输出矩阵c的数据复制到Host 端
    sdaaMemcpy(host_c, dev_c, c_size * sizeof(int), sdaaMemcpyDeviceToHost);


    memset(true_c, 0, c_size * sizeof(int));
    // Host端计算 用于对比的输出矩阵c的数据
    for (b = 0; b < B; b++) {
        for (i = 0; i < M; i++) {
            for (j = 0; j < N; j++) {
                for (k = 0; k < K; k++) {
                      true_c[b * M * N + i * N + j] +=
                          ((int)host_a[b * M * K + i * K + k])
                           * ((int)host_b[b * K * N + k * N + j]);
                }
            }
        }
    }
    // 将 从Device 端复制过到Host 端的输出矩阵c 的数据和Host端自己计算的
    // 输出矩阵c的数据做逐一对比，如有不同报错
    for (i = 0; i < c_size; i++) {
        if (host_c[i] != true_c[i]) {
            printf(
                "error: find mismatching data on %dth value,     device: %d, host: %d\n",
                i, (int)host_c[i], (int)true_c[i]);
            break;
        }
        if (i == c_size - 1) {
            printf("matmul compare finished\n");
        }
    }
    // 释放Host端内存空间
    free(true_c);
    free(host_c);
    free(host_b);
    free(host_a);
    // 释放Device端内存空间
    sdaaFree(dev_c);
    sdaaFree(dev_b);
    sdaaFree(dev_a);


    return 0;
}
```

## 7. 性能优化

算子的性能对模型训练、模型推理等均有较大影响。本文主要介绍如何基于 SDAA C 从计算和访存两个维度提升算子的性能。

**计算相关**

- 增强指令或指令中数据的并行度，例如：使用向量指令，有效利用指令的流水等，具体可以参考《性能优化手册-算子篇》。
- 增加计算数据的并行度，例如：使用多线程，多任务的并行计算，具体可以参考《性能优化手册-SDAA C 篇》。

**访存相关**

- 从时间上提高访存的利用率，例如：非阻塞型的访存数据搬运，采用双流水的方式，减少数据的等待时间；增加内存搬运硬件的利用率，具体可以参考《性能优化手册-算子篇》。
- 从数据空间上提高访存的利用率，提高数据的重复利用率，减少数据的搬运次数；通过数据广播的方式进行数据搬运，减少数据的重复搬运次数，具体可以参考《性能优化手册-SDAA C 篇》。
